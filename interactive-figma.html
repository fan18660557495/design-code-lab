01 / 问题

设计稿画了 5 态，
前端怎么写？

Variants 交付完整、颜色注释也对了，前端坐下来开始写：每种状态都做一个独立的 class？ 鼠标悬停了去切类名？—— 这是常见的两种错误打开方式。

BAD · 硬写每种状态

每个状态搞一个 class，
鼠标进出手动切

设计稿给了 5 个 Variants，前端就建 5 个 class。鼠标移上去切 btn-hover，移开切回 btn-default —— 代码冗长还容易漏。

.btn-default { background: #FF4D2E; }
.btn-hover { background: #E63D1E; }
.btn-active { background: #C4321A; }
.btn-focus { outline: 3px solid; }
 
// JS 里手动监听鼠标进出
btn.onmouseenter = () => /* 切类 */;
btn.onmouseleave = () => /* 切回 */;
问题：多写了 60% 代码，键盘 Tab 时还想不起来给 focus 也加监听 —— 无障碍直接没了。
GOOD · CSS 伪类自动接管

一个 class，
用伪类描述每种状态

只有一个 .btn，鼠标悬停 / 键盘聚焦 / 按下 / 禁用， 浏览器自动套相应的伪类。设计师交 5 个 Variants → 前端就写 5 个伪类。

.btn { background: #FF4D2E; }
.btn:hover { background: #E63D1E; }
.btn:active { background: #C4321A; }
.btn:focus-visible { outline: 3px solid; }
.btn[disabled] { opacity: .6; }
 
// JS 一行不用写
赢在哪：键盘党自动有 focus、按下自动反馈、禁用自动不能点。
5 个 Variants ↔ 5 段 CSS，1:1 直译。

03 / 关键认知

鼠标 ≠ 键盘 ——
focus 不是 hover。

新手最常踩的坑：以为 :focus 和 :hover 是一回事。 实际上它们服务的是两类不同用户，触发方式完全不一样。

🖱

Hover · 给鼠标用户

:hover

把鼠标光标移到元素上 —— 浏览器自动套用 :hover。 移开就消失。触屏设备没有 hover（手指点过去 = 点击，没有"悬停"这个动作）。

把鼠标移到按钮上 → 看变深
⌨

Focus · 给键盘用户

:focus-visible

用键盘 Tab 一个个跳到元素 —— 浏览器自动套用 :focus。 这是给看不见鼠标光标的用户用的（视障 / 不便用鼠标 / 喜欢键盘）。

点一下输入框旁边 → 按 Tab 跳到按钮 → 看紫色描边
划重点 · 设计师视角

:focus-visible 比 :focus 更聪明：它只在键盘 Tab 时显示描边，鼠标点击不会显示。 这样既照顾键盘党，又不会让鼠标用户看到讨厌的"点完按钮还有个框"。

做交付时：5 态 Variants 里给 Focus 配一个清晰可见的描边 / 边框 —— 这是无障碍最低标准。 浏览器虽然有默认 focus 样式，但常被 reset 掉，所以你要明确告诉前端"这一态需要保留 / 自定义样式"。

04 / 状态升级

loading 不是伪类，
是 React state。

hover / active / focus 是瞬态 —— 鼠标动一下就消失了。 但 loading / 选中 / 折叠这种"持续状态"，靠 CSS 伪类管不了，要用 React state。

Component · Button / Variant: state
state=default
立即购买
state=loading
提交中…
SubmitButton.tsx
const [loading, setLoading] = useState(false);   async function onSubmit() { setLoading(true); // 进入 loading await api.submit(); setLoading(false); // 自动退出 }   return ( <Button disabled={loading} onClick={onSubmit}> {loading ? '提交中…' : '立即购买'} </Button> );
点一下试试 · 2 秒后自动退出 loading
React state · loading = false
关键区分

瞬态用 CSS 伪类，持续态用 React state。 区分标准："是否需要在 JS 里手动开 / 关"。

鼠标移开 hover 自动消失 → 伪类。 但 loading 一旦进入，不调用 setLoading(false) 永远出不来 → state。 同理：选中 / 折叠 / 编辑中 / 选项卡激活 —— 全是 state。

05 / 互动 · 5 态全开

真按钮，
5 种状态真触发。

左边是一个真实按钮，右边是它的 CSS + React state。 悬停 / Tab / 按下 / 禁用 / 点击 —— 任何动作触发哪一态，对应那段代码就高亮。

default :hover :focus-visible :active [disabled] loading (state)
Button.css + Button.tsx 触发哪一行 就高亮

设计师该交什么

5 态 Variants（含 Focus！）+ 颜色注释。 特别注明：Focus 描边是否需要自定义样式（默认浏览器有，但常被 reset）。 Loading 加一个独立 Variant，画 spinner + 文案。

前端该写什么

一个 class，4 个 CSS 伪类（hover / focus-visible / active / [disabled]）。 不要给 hover / focus 加 JS 监听 —— 浏览器原生处理。 要用 React state 管 loading 这种持续态。
