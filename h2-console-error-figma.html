02 / 三种错

设计师会撞上的
就这三种

所有报错都会以 XxxError 开头。学会认这三种 —— 90% 的红字都是它们其中之一。每种配真实代码 + 报错截图 + 一句话修法。

TypeError
类型搞错了

最常见的一种。你以为变量是某种东西，但它其实是 undefined / null。比如你想 user.name，但 user 还没拿到，是空的。

遇见率 · 高
— 一段会出错的代码
const user = data.currentUser;  // 没拿到 → undefined

// 想读 user.name —— 但 user 是 undefined
console.log(user.name);
怎么修先判断一下："如果 user 存在再读 .name"。代码里写 user?.name，这个问号叫"可选链"。
— 浏览器会这么喊
●Uncaught TypeError: Cannot read properties of undefined (reading 'name') at App.jsx:8:21 at renderWithHooks (react-dom.development.js:14985) ...还有 12 行 React 内部
ReferenceError
用了不存在的东西

你提到的那个变量 / 函数，JavaScript 从来没听说过。99% 的情况：拼写错 或 忘了 import。

遇见率 · 中
— 一段会出错的代码
// 写好了 userList
const userList = ['a', 'b'];

// 但下面拼成了 userLst（少了一个 i）
console.log(userLst);
怎么修对照一下变量名。看到 "X is not defined" → 直接搜 X，看看是不是写错了字母 / 大小写。
— 浏览器会这么喊
●Uncaught ReferenceError: userLst is not defined at App.jsx:6:13 at HTMLButtonElement.<anonymous> (...)
SyntaxError
句子写飞了

代码本身的语法不对 —— 少了括号、少了引号、多了逗号。这种错通常更狠：整个文件一行都跑不了，整个页面直接白屏。

遇见率 · 中低
— 一段会出错的代码
const user = {
  name: 'Alice',
  age: 30
  // 少了闭合的 }

console.log(user);
怎么修看错误指向的那一行 —— SyntaxError 通常报得有点偏（指到文件末尾），但你只要在那附近找 { } ( ) " ' 是不是配对就好。VSCode 会自动配对，AI 也会。
— 浏览器会这么喊
●Uncaught SyntaxError: Unexpected end of input at App.jsx:12
套路看到 XxxError → 先认是哪一种 → 后面那句 "is not defined" / "Cannot read..." / "Unexpected..." 就是它在告诉你具体哪不对。三种连起来读，像看天气预报一样自然。

04 / 主动问

console.log
是设计师的朋友

前面三节都在讲"代码自己喊出来的报错"。但还有一招更主动 —— 让代码"说话"。任何变量后面加一句 console.log(变量)，它就会把当时的值打到 console 里。设计师也能用。

关键不是只有工程师才能加 log。你跟 AI 说"在 X 行加 console.log 看 user 是什么"，它就会加。debug 一大半就是主动问，不是"等错出来"。
console.log()

基本款 · 看一个变量

最基础的一个。括号里塞什么，console 就吐什么。

const user = getUser();
console.log(user);  // ← 加这一句
— Console 输出
▸ {name: "Alice", age: 30, role: "designer"}
console.table()

数组款 · 看一组数据

变量是个数组（list） / 一堆对象的时候用它，会自动排成表格。

const users = getUsers();
console.table(users);  // ← 排成表格
(index)nameage
0"Alice"30
1"Bob"25
2"Cara"28
console.error()

高亮款 · 让自己的 log 标红

想让自己加的 log 在一片白字里特别显眼？用 error，浏览器会给它打上红色 ❌ 图标 —— 跟真报错一样。

if (!user) {
  console.error('user 没拿到！');
}
— Console 输出（红框）
● user 没拿到！
console.log() with label

加标签款 · 一次看多个

想看好几个变量？前面加个字符串当标签，找起来就清楚多了。

console.log('[user]', user);
console.log('[loading?]', isLoading);
console.log('[response]', response);
— Console 输出
[user] {name: "Alice"}
[loading?] false
[response] undefined ← 看，问题在这
↳ 让 AI 帮你加 LOG · COPY-PASTE

不会写也没关系，
你会"指挥 AI 加 log"就够了

AI 写完代码 → 你跑起来不对 → 不要立刻让它"修一下"。先让它加 log，搞清楚到底哪一步出问题。这是个万能 prompt：

"这段代码跑起来 user.name 报 undefined。
请在 UserList.jsx 第 14 行 之前 加一句 console.log('[user]', user)，
我跑一遍把 console 的输出贴回来给你，再决定怎么改。"
05 / 红有三种

看到红色，
先看图标

Console 里"看上去是红色 / 警告"的东西其实分三层。真红 才是必须修的；黄色 是提醒；灰色 info 是浏览器自己唠叨。看图标比看颜色准。

真红 · Error
red ● — code is broken

代码挂了。当前这个功能完全跑不动 —— 按钮点了没反应、组件渲染失败、页面白屏。

必须修 · 先看错误类型 + 第一行 stack
警告黄 · Warning
yellow ▲ — heads up

能跑但有隐患。比如"你这个 prop 传错类型了"、"key 重复了"、"React 18 这种写法即将废弃"。

可以晚点处理 · 不影响当前演示
Info 灰 · Info
grey ⓘ — fyi

浏览器自己唠嗑。"cookie 已过期"、"加载完成"、"开发模式开启"、"hot reload 触发"。

90% 可以无视 · 偶尔看看就行

06 / 速查卡

三张卡片
塞进口袋

所有的精华浓缩成三张卡。打开 console 看到红字 —— 按这个顺序读，几乎不会卡。

PART 01 · 类型

认 3 种 Error

XxxError 开头
TypeError
变量是 undefined / null，但你当成对象在用。
ReferenceError
变量根本不存在 —— 拼错了 / 忘 import。
SyntaxError
语法不对 —— 少括号 / 引号 / 逗号。整文件挂。
PART 02 · 顺序

三件套阅读法

read in 3 steps
STEP 1
第一行红字 — 错误类型 + 错误信息。
STEP 2
下面 stack 第一行 — 文件名 + 行号。指向你的代码。
STEP 3
剩下全部 跳过。那是 React / 浏览器内部，不归你管。
PART 03 · AI

看不懂就问 AI

copy-paste prompt
第一招 · 全贴
把整段红字 + "我刚做了什么" 一起喂给 AI。
第二招 · 加 log
"在 X 行前加 console.log 看 Y 是什么"。
第三招 · 不要直接修
先让 AI 解释错在哪、为什么。看懂再改。
