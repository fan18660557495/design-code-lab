阴影 / 层级
Shadow & Elevation

设计稿里的 shadow 不只是"一坨黑色模糊"。它在表达表面离背景多远、光从哪来、哪个面板压在上面。翻成 CSS 时，最重要的事是把"视觉层级"翻译成可复用的 shadow token，而不是逐个 surface 手调一组数字。

目标层级 / 深度
心智模型Surface above canvas
CSS 属性box-shadow / filter
结论先定级别，再定数值
level 01 · card

贴近背景

轻边框 + 极浅阴影，给普通卡片和列表项。

level 03 · popover

抬起来

两层阴影：贴边层接触感，扩散层空间感。

anti-pattern

脏黑块

只把黑色加深，界面只是脏，不一定更有层级。

01 / 先拆设计语言

阴影不是装饰，
是 空间关系。

Figma 里你看到的是 x、y、blur、spread、color。前端真正要还原的是：这块 surface 是 flat、raised，还是 overlay。先回答这个，再决定数值。

WRONG MODEL

把每个阴影都当独立特效

每张卡片单独配一组 shadow，页面很快会出现 12 种"差不多但都不一样"的层级。维护时找不到统一口径。

.card-a { box-shadow: 0 9px 31px #0003; } .card-b { box-shadow: 2px 14px 40px #0004; }
RIGHT MODEL

先定级别，再套 token

把效果从"这张卡片的影子"升级成"第 3 层表面"。以后组件只选层级，不重新发明阴影。

:root { --shadow-card: 0 1px 2px #16161d14, 0 8px 24px #16161d1c; --shadow-popover: 0 1px 2px #16161d1f, 0 18px 40px #16161d24; }
01判断表面flat / raised / overlay，不同表面不该用同一套影子。
02统一光源全站 y 方向和 x 偏移要一致，否则页面像多盏灯乱照。
03拆成两层ambient 贴边接触，directional 扩散表达高度，别只靠一个大 blur。
04落成 token组件写 var(--shadow-popover)，而不是复制 Figma 数字。

03 / 设计到代码对照

        
Figma 数字，
不要 逐个照抄。

      

      

        Figma 的 Effect 是设计表达，CSS 的 shadow 是系统实现。两边单位相似，但维护方式不一样。
      

    


    

      
        
设计稿看到代码里先问推荐翻译

      
      
        
多张卡片都有轻阴影
参数略有不同		它们是同一类 surface 吗？合并成 --shadow-card，不要保留 N 种"差不多的卡片"。

        
弹窗阴影特别重
设计想强调遮挡关系		是不是 overlay 层？背景是否需要 dim？shadow + backdrop，而不是无限加黑。

        
异形元素有影子
圆角、凹角、票券形状		阴影是否需要跟随透明轮廓？用 filter: drop-shadow()，而不是 box-shadow。

        
暗色模式阴影变淡
对比度变化		暗背景上还需要阴影吗？暗模式常常改用 border + 微高光，不再依赖深色阴影。

      
    

  






  

    

      

        
04 / 交付口径

        
给 AI 和前端的
shadow spec。

      

      

        描述阴影时不要只说"加点阴影"。要把层级、场景、token 和限制说清楚，AI 才能给出可复用的实现。
      

    


    

      

        01 / LEVEL
        
用级别说话

        
"这里是 popover，不是 modal。"级别比具体数值更稳定。

        
  • base：无阴影
  • card：轻浮起
  • popover：明确分离
  • modal：压住所有

      

      

        02 / TOKEN
        
把效果命名

        
效果 token 可以跟颜色、圆角、间距一样进入设计系统。

        
  • --shadow-card
  • --shadow-popover
  • --shadow-modal
  • --shadow-toast

      

      

        03 / TEST
        
换背景检查

        
浅、深、彩色背景都要看一眼。阴影不是只在设计稿那块画布上成立。

        
  • 不要脏
  • 不要糊
  • 不要抢内容
