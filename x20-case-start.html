区域 1 活跃
Q1 / 5 · 一致性 · 字段

"活跃" chip 的颜色 / 文案 / 阈值

这个 chip 的颜色、文案、"活跃"的阈值是谁定的？

颜色来自 design token——color/success/500。不是设计师拍脑袋选的绿色。
组件来自 design system——<StatusBadge variant="success" />。所有"活跃"用同一个组件。
阈值来自 业务字典——"30 天内有过交互 = 活跃"。这条规则写在业务字典，不是组件里。
⚠ 陷阱：设计师在 Figma 里直接选了 #28ca42 → 工程师写死在组件 CSS → 主题切换时整个 UI 错乱 / 业务说"改成 60 天活跃"时全站漏改。
涉及：X18 一致性 · X10 字段意识

138****1234
Q2 / 5 · 角色 · 字段

电话脱敏 138****1234

"中间 4 位打码"在哪一层决定？格式由谁定？

脱敏后端——返回时根据当前用户角色决定要不要打码。前端拿到的就是 138****1234，不是完整号码。
显示格式UI 层决定空格 / 连字符——但不决定打不打码。
规则销售看完整 · 客服看打码 · 客户本人看完整——同一字段 3 种返回值。
⚠ 陷阱：设计师只画"打码态"+ 写一句"电话脱敏" → 工程师在前端用 str.replace 写正则 → 抓包能看到完整号码 → 合规事故。脱敏必须发生在后端。
涉及：X4 角色权限 · X10 字段意识

Q3 / 5 · 对象 · 引用

关联商机列表

这 3 条商机是客户的字段吗？

不是字段商机是独立对象，不是客户上的 opportunities: [] 字段。
关系1:N——1 个客户对应多个商机；每个商机有自己的 customer_id 反向挂回来。
接口详情页拉客户主对象时不一起拉商机——分页拉 GET /opportunities?customer_id=123，否则 1 个超级客户能拖死页面。
⚠ 陷阱：设计师把"商机列表"画成客户卡里的一个字段 → 工程师真的在 customer 表加 JSON 字段 → 改一条商机要更新整个客户对象 → 数据漂移 + 性能崩盘。
涉及：X1 对象不是页面 · X2 字段·关系·引用
区域 4 + 新建商机
Q4 / 5 · 角色 · 任务流

"+ 新建商机" 按钮

谁能看到这个按钮？点了之后去哪？

可见性仅 Sales 角色 + 客户为 active 状态。客服 / 财务看不到；客户 dormant 时按钮 disabled。
权限来源后端返回 actions: ["create_opp"] 数组；前端按数组渲染——不是前端硬编码 if (user.role === 'sales')。
跳转点击 → 跳 /customers/123/opportunities/new，URL 里带 customer_id 预填——不是 modal。
⚠ 陷阱：设计师写"管理员能看" → 工程师在前端写 if (role === 'admin') → 加新角色时全站漏改 + 抓包能看到所有按钮。权限永远是后端下发的能力列表。
涉及：X4 角色权限 · X5 任务流
区域 5 删除
Q5 / 5 · 状态 · 异常 · 演进

"删除" 按钮

点了之后真的"删掉"客户吗？关联商机怎么办？

可见性仅 Admin 角色，且客户必须是 archived 状态（active 客户不能删）。
交互点击 → 二次确认 modal「输入客户名确认」——不能用普通 confirm() 一键删。
数据软删除——置 deleted_at = now()，不真删行。审计日志保留 5 年。
级联关联商机 / 合同 / 活动不删，标记 customer_deleted = true，用于历史查询和报表。
⚠ 陷阱：设计师只画一个"删除"按钮 + tooltip"删除客户" → 工程师写 DELETE /customers/123 硬删 → 客户删了，商机变孤儿，财务报表对不上去——删除是状态机迁移，不是物理删。
涉及：X4 + X7 异常 + X8 状态机 + X19 交付

一张稿能数出来 12 个区域，但每个区域背后藏 5 个隐藏决策（来源 / 阈值 / 权限 / 状态 / 组件）。

看见的 12 = 真实的 60。

iii

案例三部曲是练习场

下一篇 X21 拆 4 张图（对象 / 流程 / 状态 / 组件树）；X22 组装成 spec。

稿子之外的功夫，才是真正的设计。

划重点 · TL;DR

看到稿，第一句不是"好看吗"——是"10 维度都识别完了吗"。每一个区域都是一个决策入口，背后都是规格、权限、状态、契约。

X1 教你识别对象，X2-X19 教你拆维度。X20-X22 这三篇是把所有维度合到一张稿里看的练习场——拿这张张明详情页练手，下一张甲方稿你的眼睛就不一样了。
