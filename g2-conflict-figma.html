01 / 起源

冲突
是怎么产生的

本质就一句话：同一行代码，两个人都改了，git 不替你做主。 来看一遍这件事的发生过程 —— 别急，按顺序点下面的按钮。

动画主线是黑色那条 main。两个分支从 main 分出去：你的（橙）和 ta 的（蓝）。两边都改了同一行 —— 合并的时候 boom。
按顺序点 ›
› 点上面的按钮看动画。
main · 主线 你的分支 ta 的分支 冲突点
关键注意 git 没"出错"。它做了它能做的事 —— 把 ta 的改动合进 main。 你再合的时候，main 第 12 行是蓝色，你的第 12 行是橙色 —— git 不替你乱选，它把两份摞起来给你看，让你来定。
02 / 标记

那一片
红字到底是什么

<<<<<<< ======= >>>>>>> 不是乱码 —— 是 git 把两份草稿摞起来给你看的标记。三段，一段一段拆开看就懂。

互动把鼠标放到右边的解释上 → 左边代码对应那一段会高亮。
Button.tsx · 冲突中 
<<<<<<< HEAD
  color: '#ff5e1a',  // ← 你这边
=======
  color: '#1a8fff',  // ← ta 那边
>>>>>>> feature-blue
三个尖括号 · MARKER
这三行是 git 加的"分割线"

<<<<<<< 表示"上面的草稿开始"，======= 是分隔线， >>>>>>> 表示"另一份草稿结束"。 解冲突 = 删掉这三行，加上你想保留的内容。

第一段 · 你这边 (HEAD)
"HEAD" 就是你当前的版本

上半段（橙色）是你正在工作的代码。git 把它叫 HEAD，意思是"你脑袋里这一份"。

第二段 · ta 那边
是要合进来的别人的版本

下半段（蓝色）是你正打算合进来的另一个分支。结尾那行 feature-blue 就是分支名 —— git 直接告诉你"冲突来自哪个分支"。

记住解冲突的全部操作 = 编辑这个文件，删掉那三行尖括号 + =======， 留下你想要的内容。然后保存、git add、git commit。没了。

04 / 习惯

        
怎么少遇到冲突
三个工作习惯

      

      

        冲突永远不会消失（多人协作就一定会有），但 90% 的冲突可以靠这三件事避免。
        每条都配一个反例 —— 不这么做会怎样。
      

    


    

      

        
HABIT · 01

        
开干前先 pull 一下

        
git pull # 把别人的改动先拉下来

        
每天开始干活前、每次准备改一个新文件前，先 git pull 一下。把别人这段时间的改动先同步到你这边 —— 你后面动的就是最新的代码。

        

          不这么做你昨天 fork 出来一份，今天直接接着改。等一周后想合回去 ——
          整周的差距都堆在那里，冲突会非常多。
        

      


      

        
HABIT · 02

        
改之前在群里说一声

        
// "我现在改 Button.tsx 哈"

        
动手改一个共享文件前，在 Slack / 飞书 / 微信群里说一句"我在改 X 文件"。同事看见了就会绕开 —— 这是最低成本的协调，比解冲突省事一百倍。

        

          不这么做你和 ta 各自闷头改同一个文件 2 小时。
          合并的时候发现两个人改了 80% 同样的地方 —— 白干一个人的活。
        

      


      

        
HABIT · 03

        
小步 commit · 小步合

        
git commit -m # 每改一件事就提交

        
一次只做一件事，做完就 commit + push。每个 commit 越小，冲突面积越小。改完颜色就提交、改完文案再提交，别攒一大坨。

        

          不这么做三天没 commit，攒了 80 个改动一起提。
          冲突起来会跨越好几个文件、几百行 —— 解到怀疑人生。
        

      

    

  







  

    

      

        
05 / 反悔

        
真的搞砸了？
还能反悔

      

      

        万一冲突太复杂、你改着改着把自己绕晕了 —— 别慌。
        git 几乎所有操作都可以撤销。这是设计师最该记住的一件事。
      

    


    

      反悔三连点下面三个步骤，看每一步的命令是什么、做了什么。
    


    

      

        

          
STEP 01

          
git status

          
先看现状

          
git 告诉你"现在哪几个文件冲突中"。

        

        
→

        

          
STEP 02

          
git merge --abort

          
一键撤销

          
回到没冲突前的样子，就像没合过。

        

        
→

        

          
STEP 03

          
// ask a friend

          
叫工程师同事

          
让懂的人陪你看一眼，5 分钟搞定。

CARD 01 · 标记

        
三段标记

        
看到这个先别关页面

        

          
上半段

          
<<<<<<< HEAD 之后是你的版本。

          
分隔线

          
======= 是中间的隔板。

          
下半段

          
>>>>>>> 分支名 之前是别人的版本。

          
解法

          
删掉这三行 + 留下你要的内容。

        

      


      

        
CARD 02 · 选择

        
三种选法

        
你只需要选一个

        

          
保留我的

          
留 HEAD 那段，删 ta 的。
git checkout --ours 文件名

          
保留 ta 的

          
留 ta 那段，删我的。
git checkout --theirs 文件名

          
两个都要 / 手动改

          
直接打开文件编辑 —— IDE 会让这件事很简单。

        

      


      

        
CARD 03 · 反悔

        
救命三连

        
不会做就 abort

        

          
看现状

          
git status · 看哪些文件冲突。

          
撤销整个 merge

          
git merge --abort · 像没发生过。

          
找人

          
叫工程师同事 ——不丢人，是常态。
