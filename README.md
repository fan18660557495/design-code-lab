# Design × Code 实验室

> 设计师该懂多少代码？看懂界面是怎么被代码实现的 —— 把代码侧最基础的几件事讲清楚。

给设计师的「Code × Design」科普合集。用「小游戏 + Figma 类比」把代码侧最基础的几件事讲明白 —— 学完你不一定会写代码，但当 AI 不听话、当你跟前端讨论改动时，**能说得出口、听得懂、知道怎么改**。

**在线阅读：** https://fan18660557495.github.io/AI-Labbook/

## 为什么做

我自己最早用 AI 做 UI 的时候，经常一脸懵 —— AI 生成的稿子看着差不多，但拆开就错；想让它改，又说不清；改完哪儿都不对劲，又指不出问题在哪。

后来跟很多设计师朋友聊，发现大家都有这种挫败感。本质问题其实很简单 —— 我们对 AI 在干嘛、代码在做啥，几乎一点不懂；但纯文字的代码教程，又看不下去也记不住。

所以做了这个合集，想把设计和代码之间的**鸿沟**，弄小一点。本质上，这是一座桥，或者说一份**翻译**。
