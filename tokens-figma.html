01 / 问题

散装颜色，
谁都不知道
哪个是"那个橙"。

设计师 A 复制粘贴了 #ff4d2e，设计师 B 用了 #ff5733。 前端打开你的 Figma，从五个相似的橙里挑一个。 改品牌色那天，没人知道要改哪几处。

BAD · 散装色码

每个图层都是一个"裸值"

颜色直接刷在每个元素上。文件里有五个相似的橙色，你也分不出哪个是品牌色、哪个是临时记的。

#ff4d2e 主按钮
#ff5733 徽章
#ff4f30 链接 hover
#fe4c2d 图标
真实场景：老板说"把品牌色调蓝"。前端问你：要改哪些颜色？你打开稿子，开始一个一个找。
GOOD · 命名色板

一个名字，一处定义

每个颜色都有名字，比如 brand/primary。所有用到品牌色的地方，都引用这个名字 —— 而不是色码。

brand / primary → #ff4d2e
brand / primary → #ff4d2e

brand / primary → #ff4d2e
真实场景：老板说"把品牌色调蓝"。你只改 brand/primary 一个值，所有按钮、徽章、链接、图标全部跟着变。
02 / 命名

不是说 #ff4d2e，
而是说 brand/primary。

Figma Variables 干的事，本质就一句话：给一个值起一个名字。
从此以后，设计稿和代码都用名字交流，不再用色码。

左边的 Figma 面板，右边的 CSS 文件，其实是一份字典

鼠标移到任意一行 ——brand/primary 在 Figma 里叫这个名，到了 CSS 里叫 --brand-primary。 名字略有不同（连字符 vs 斜杠），但它们指向同一个值。 这就是设计稿和代码"对得上"的起点：约定一套共同的名字。

C0 · TOKENS

色板是给前端的礼物。

这里所有的"主色"——Logo 底色、按钮、徽章、链接 ——都引用了同一个 token。改 token，全部跟着变。

当前 brand / primary
--brand-primary
#ff4d2e
点一个色板换它
/* 同步生效中 */ --brand-primary: #ff4d2e; --brand-soft: #ffe4dc;
一次点击，6 处元素同步换色 —— 你没改任何具体颜色，只改了一个 token。
05 / 实践

什么时候
该建一个 token？

简单的判断：这个值出现在两个以上的地方 —— 就该建 token。 下面是设计师每天会遇到的四类清单。

01

颜色 · Color

品牌色、文字色、背景色、边框色、状态色（成功 / 警告 / 错误）—— 全部建 token。brand-primary、text-ink、bg-paper。

02

字号 · Type

不是每个 px 数字都要建，而是建一套"阶梯"：display / heading / body / caption。改字号阶梯，全站排印一起调。

03

间距 · Spacing

4 / 8 / 16 / 24 / 48 这种 4 倍数阶梯。Auto Layout 的 padding 和 gap 用它们，不要手敲 13、17 这种零碎值。

04

圆角 · Radius

容易被忽略，但同样关键。radius-sm = 4，radius-md = 8，radius-pill = 999。圆角风格统一感全靠它。

BAD · 用值命名
取名取的是颜色本身
orange-500 改成蓝色就尴尬了
red-bright 名字限死了颜色
color-1 完全看不出用在哪
GOOD · 用语义命名
取名取的是用途
brand / primary 改成蓝色也合理
action / cta 说清楚干什么的
status / danger 语义跨颜色稳定

记住一句话：token 的名字应该说明 "这是什么用途"，而不是 "这是什么颜色"。 因为颜色会变，用途不变。brand-primary 这个名字，明天换成蓝色也还合理；orange-500 换成蓝色就成笑话了。
