01 / 问题

设计稿和代码
到底怎么对应？

你做完设计稿丢给前端，开发出来的 UI 看着像，但每改一个字段，所有人都要忙半天。 问题不在「沟通」，问题在「结构」。

BAD · 各管各的

设计稿 ≠ 代码

设计稿里你画了一张漂亮的卡片。前端拿到稿子，重新拆、重新搭、自己起名字。 于是同一件东西在两个世界里有两种结构、两套名字。

结果：你说"改一下卡片上的标题样式"，前端要在 5 个地方改，每个还都不一样。

GOOD · 同一件事

设计稿 = 代码结构

你画的 Frame 是什么，前端代码里就有什么；你的图层叫什么名，代码组件就叫什么名。 一一对应、一处改、处处变。

结果：你说"卡片标题改成大一号"，前端只动一处。

" 设计稿不是图，是结构。 画图层 = 在搭一棵会被代码"翻译"的树。你怎么搭，前端怎么写。

03 / 复用

同一个组件，
用 N 次。

Figma 里你画一个 Card Component，从 Asset 拖出来 N 个 Instance。 代码里前端也只写一次，调用 N 次。拖动滑块试试。

◆

{ Title 标题 }

{ Price 价格 }

这是 Figma 里的 主组件 · Main Component。 所有 Instance 都从它复制出来。它定义了"骨架"： 上面一张图、中间一行标题、下面一个价格 —— 三块的位置和样式固定。

改主组件的圆角、配色、字号、间距 → 所有 Instance 自动跟着变。
代码侧也是同样的逻辑：前端写一次 Card，然后调用无数次。
Instance 数量
3
画 1 张和画 20 张 —— 代码量一样。
代码侧的"循环写法" — 写一次，自动生成 3 张
// products = [{...}, {...}, {...}] ← 一组数据，可多可少
<Grid>
products.map(item =>
<Card image={item.image} title={item.title} price={item.price} />
)
</Grid>
" Figma 改主组件 → 所有 Instance 跟着变。
代码改一处 → 所有调用同步更新。 这就是为什么前端反复说："这个东西要做成可复用的组件"。和你在 Figma 里把卡片做成 Component，是同一回事。
04 / 数据 vs 样式

什么是变的，
什么是固定的。

一张卡片里，"会变"的东西（图、文案、价格）是数据； "固定"的东西（圆角、配色、字号、间距）是样式。 切换下面的开关，看清这两层。

圆角 · 阴影
图 · IMAGE

标题 · TITLE 手冲咖啡 · 哥伦比亚

价格 · PRICE ¥ 38

字号 · 颜色 柑橘香气 · 中度烘焙 · 醇厚回甘

数据层 · 会变的

  • 图片 — 不同商品有不同图
  • 标题 — 不同商品有不同名字
  • 价格 — 每张卡片不一样

设计师改文案 / 换图 = 改数据。前端会把这些字段做成"参数"留出来，外部传什么，卡片就显示什么。

样式层 · 固定的

  • 圆角 / 阴影 — 整套卡片统一
  • 背景色 / 字号 / 间距 — 设计规范

设计师改颜色 / 改间距 = 改样式。这些写一次就好，所有卡片自动跟着变（就像 Figma 里改主组件）。

" 想清楚什么会变、什么不会变，
你的设计稿就有"骨架"了。 前端拿到稿子第一件事就是问这个。你提前想好 → 一拍即合。
05 / 命名

图层名 =
代码组件名。

Figma 图层叫什么名，代码侧组件就叫什么名。 随手命名 vs 有意义命名 —— 切换看看差距。

BAD · 前端只能猜

"Frame 247 / Group 3 / Rectangle 12"

▾ ▢Frame 247
├ ▢Group 3
│ ├ ○Ellipse 8
│ └ TText 12
├ ▢Rectangle 47
└ ▢Group 12
├ TText 18
└ TText 19

前端打开稿子，每个图层都靠猜。"Group 3 是头像区？还是按钮组？" 猜错就要返工。命名出来的代码也是 div247 / group3 / rect47，没人看得懂。

GOOD · 一眼看懂

"ProductCard / Header / PriceTag"

▾ ▢ProductCard
├ ▢Header
│ ├ ○Avatar
│ └ TUserName
├ ▢CoverImage
└ ▢PriceTag
├ TCurrency
└ TAmount

前端读一遍图层名，整个页面结构就清楚了。 代码组件直接用同名：<ProductCard> · <PriceTag>。 以后改东西，定位、沟通、维护，全部走在同一个词汇表上。

送给前端的礼物

起好图层名，是设计师能给前端最重要的第一份礼物。
五分钟改名字 → 省下半小时来回沟通。

06 / 速查

出师 ·
下次看稿子，
记住这 4 句话。

不用记代码，只要记住这四个 Figma 视角的小习惯， 你的稿子就比 80% 的设计师更好对接。

01

图层嵌套 = 组件树

你怎么嵌 Frame，前端就怎么嵌组件。Figma 的图层面板就是代码的目录。

02

Component / Instance

同一种东西出现多次 → 一定要做成组件。一处改，处处变。

03

数据 vs 样式

会变的（文字 / 图 / 价格）= 数据；固定的（颜色 / 圆角 / 间距）= 样式。

04

命名要"有人话"

"ProductCard" 比 "Frame 247" 好。前端能看懂的图层名 = 干净的代码。
