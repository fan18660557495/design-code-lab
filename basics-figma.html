01 / 问题

设计稿到代码
中间的"黑盒"

你交完稿，前端默默工作几天，网页上线了。 中间到底发生了什么？知道一点结构，比完全不知道，沟通效率会差一个数量级。

BAD · 完全黑盒

"反正我交了稿，剩下让前端弄"

能做出来当然好，但只要稿子和成品对不上、需要改一下间距、想加个 hover 状态 —— 你完全不知道哪句话能说清楚需求。

你：稿子 → ？？？ → 网页
// 中间是一团迷雾
// 出问题时无法精确定位
GOOD · 知道大概结构

"代码 = 三件套，每件管一件事"

不需要会写。但你知道：HTML 管"有什么"，CSS 管"长什么样"，JS 管"动起来"。沟通改稿、提需求、看 AI 生成的代码 —— 都顺多了。

02 / 骨架

HTML 是骨架
= Figma 图层树

Figma 左侧那棵图层树 —— Frame 套 Frame 套 Text/Image —— 翻译成代码就是 HTML。 每一层"图层"对应一个"标签"。结构是一一对应的。

互动点左侧 Figma 图层 → 右侧 HTML 对应行会高亮。反过来点代码也一样。
图层 · LAYERS
▾ ⬚ Card
▾ ⬚ Header
· T Title
· T Description
· ▭ Button
card.html 
<div class="card">
  <div class="header">
    <h3>标题</h3>
    <p>一段描述...</p>
  </div>
  <button>了解更多</button>
</div>
Figma 视角
一棵图层树。Frame 套 Frame，每个图层有名字、有层级、有缩进。
代码视角
一棵标签树。<div> 套 <div>，每个标签有名字、有层级、有缩进 —— 一模一样的事。
03 / 装修

CSS 是装修
= Figma 属性面板

Figma 右侧那一排属性面板 —— 颜色、字号、间距、圆角 —— 翻译成代码就是 CSS。 HTML 决定"有什么"，CSS 决定"长什么样"。

互动点左侧 Figma 属性行 → 右侧 CSS 代码对应位置会高亮。
Layout · 布局
P 内边距
⌒ 圆角
Fill · 填充
FF4D2E 100%
Text · 文字
Aa 字号
FFFFFF 字色
style.css 
.button {
  background: #ff4d2e;
  color: #ffffff;
  padding: 20px;
  border-radius: 8px;
  font-size: 14px;
}
Figma 视角
选中一个图层，右侧属性面板就是它"长什么样"的全部信息。
代码视角
一组 属性: 值。属性名几乎跟 Figma 一样：padding = 内边距、border-radius = 圆角。
04 / 行为

JS 是行为
= Figma Prototype

Figma Prototype 里那根连接两个 Frame 的箭头 —— 点击 / hover / 切换状态 —— 翻译成代码就是 JS。 前两件管"静态长什么样"，JS 管"用户操作时怎么响应"。

Frame · Default
深色模式
On Click → Smart Animate
Frame · Dark
深色模式
script.js 
// 选中那个按钮
const btn = document.querySelector('button');

// 监听"点击"这个动作
btn.addEventListener('click', () => {
  // 切到深色样式
  card.classList.toggle('dark');
});
Figma 视角
Prototype 连线：两个 Frame，中间一根箭头，注明"On Click"、动画类型。
代码视角
事件监听：盯住某个元素，等"click"发生，然后改一下样式 —— 完全同一件事。
注意页面上看到的所有"动起来"的部分 —— 弹窗、tab 切换、表单校验、菜单展开 —— 背后都是 JS 在响应。Figma 里看不到的"用户操作之后"，都靠它。

PART 01 · 骨架

HTML

HyperText Markup Language
负责
页面上有什么。标题、段落、按钮、图片这些"东西"。
类比 Figma
图层树。Frame 套 Frame 的那个面板。
设计师交什么
清晰的图层结构和命名。Auto Layout 比绝对定位好十倍。
PART 02 · 装修

CSS

Cascading Style Sheets
负责
页面长什么样。颜色、字号、间距、圆角、阴影。
类比 Figma
右侧属性面板。Fill / Stroke / Effects / Text。
设计师交什么
统一的设计变量（颜色、字号、间距）和清晰的设计稿。
PART 03 · 行为

JS

JavaScript
负责
用户操作时怎么响应。点击、hover、切换、加载新数据。
类比 Figma
Prototype 连线和 Smart Animate。
设计师交什么
每个交互的所有状态：默认、hover、点击中、禁用、加载、错误...
