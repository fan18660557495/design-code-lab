---

## 一、为什么 AI 会"看不懂"你的稿子？

简单粗暴讲：**AI 看到的是像素，不是图层。**

你打开一份设计稿，眼睛扫一眼就知道：

- "这是一个卡片"
- "卡片左边是头像，右边是名字和职位"
- "中间的间距大概是 12"
- "整张卡片有 16 的内边距"

但 AI 看不到这些。它只看到一堆色块在屏幕上的某个位置。

如果你的稿子图层乱七八糟、用绝对位置硬拽出来——

AI 只能猜。**它猜的结果，就是你看到的那个"差点意思"。**

---

## 二、关键转折：Auto Layout = CSS Flexbox

这是我研究了三天才反应过来的事：

> Figma 的 Auto Layout，**不是 Figma 自己发明**的概念。
>
> 它和前端的 CSS Flexbox，**几乎一一对应**。

什么意思？

你在 Figma 里点的每一个属性，前端那边都能找到一个**对应的 CSS 写法**。

📷 *配图：Figma Auto Layout 面板 vs CSS 代码 一一对应*

| Figma | ↔ | CSS |
|---|---|---|
| Direction · Horizontal | ↔ | `flex-direction: row` |
| Direction · Vertical | ↔ | `flex-direction: column` |
| Padding | ↔ | `padding` |
| Spacing (gap) | ↔ | `gap` |
| Align (cross axis) | ↔ | `align-items` |
| Justify (main axis) | ↔ | `justify-content` |
| Hug contents | ↔ | `width: fit-content` |
| Fill container | ↔ | `flex: 1` |
| Fixed (e.g. 200) | ↔ | `width: 200px` |

**九个字段，覆盖 Auto Layout 90% 的使用场景。**

我第一次看到这个对应关系的时候，整个人都不好了——

原来我每天在 Figma 里点的那些按钮，**就是在写 CSS**。

---

## 三、再往底层看一层：每个元素都是一个盒子

这件事 CSS 圈早就讲烂了，但很多设计师可能没听过：

> **在 CSS 里，每个元素，都是一个盒子。**

每个盒子由四个东西组成。从里到外：

1. **content（内容）**——盒子里装的东西
2. **padding（内边距）**——内容到边框的距离
3. **border（边框）**——盒子的边线
4. **margin（外边距）**——这个盒子到下一个盒子的距离

📷 *配图：盒子 4 件套同心方框拆解图*

打个比方——

> 这就像一幅装裱好的画。
>
> 画本身是 content，  
> 画到画框之间的留白是 padding，  
> 画框是 border，  
> 这幅画到墙上下一幅画之间的距离是 margin。

**CSS 里 90% 的"调间距、改大小"，玩的就是这四个东西。**

而在 Figma 里——

- Auto Layout 的 padding 就是 CSS 的 padding
- Auto Layout 的 gap 就是 CSS 的 margin（在 flexbox 上下文里其实是 gap）

**同一件事，两种说法。**

---

## 四、宽度的三种玩法：Hug / Fill / Fixed

这是另一个 Figma 和 CSS 完全对齐的设计：

每个 Auto Layout 容器或者元素，宽度可以选三种模式——

- **Hug**（包住内容）— 多大就多大
- **Fill**（撑满容器）— 能多宽是多宽
- **Fixed**（固定数值）— 写死的

来看一个真实的场景：

你做一个卡片，里面放头像、名字、按钮。

- **头像** 用 Fixed 40px（必须固定）
- **名字** 用 Fill（剩下的空间都给它）
- **按钮** 用 Hug（包住"购买"两个字就行）

窗口变窄的时候，谁会让步？

**只有 Fill 在缩，Hug 和 Fixed 都不动。**

📷 *配图：窗口缩放时，Hug Fill Fixed 三种模式的反应对比*

这个机制 Figma 和 CSS Flexbox 是**一模一样**的。

你在 Figma 里设了 Fill，前端那边写的就是 `flex: 1`。

你设了 Hug，前端写的就是 `width: fit-content`。

## 五、所以怎么让 AI 真的能用你的稿子？

研究了三天，我最后的结论很简单——

> **稿子越接近 CSS 的逻辑，AI 还原越准。**

具体三条建议：

### 1. 稿子全部用 Auto Layout 包好
不要手动定位、不要"绝对位置"。

AI 看到一个 Auto Layout，它知道这是 flex 布局；看到一堆漂浮的图层，它只能瞎猜。

### 2. 图层命名清晰
- ✅ `Card / Avatar / UserName / Button`
- ❌ `Frame 12345 / Group 89 / Rectangle 7`

AI 一眼看懂结构，比你解释半天有用。

### 3. 间距用 4 / 8 / 12 / 16 / 24 这种节奏
不要一会儿 7 一会儿 13。

设计 token（设计系统的间距规范）就是这个节奏，AI 可以直接对应到 Tailwind 的 `gap-2` `gap-3` `gap-4` 这种类。

**这三条做到，我自己实测：AI 还原成功率从 50% 提到 90%。**

| 旧认知 | 新认知 |
|---|---|
| Auto Layout 是 Figma 的功能 | Auto Layout 就是 CSS Flexbox |
| 设计稿和代码是两个世界 | 你点 Auto Layout 就是在写 CSS |
| AI 还原不准是 AI 不行 | AI 还原不准是稿子不"代码友好" |
