01 / 一个登录页

三种喂法
三种结果。

同样一个登录页,给 AI 不同颗粒度的积木 —— 三种喂法,决策点 / 出错率 / 还原度 完全不同。

A
小颗粒 · 200 原子
AI 决策点
~35 次选择
每一步都从 200 个原子里挑一个 + 决定怎么排
· Input · Input · Label · Label · Label · Button · Container · Divider · Link ... 25+ 元素手工排版
结果:Layout 偏移、间距不一、按钮跟稿子高 4px。 还原度 ~ 60%
B
中颗粒 · 30 区块
AI 决策点
~5 次选择
从 30 个复合组件里挑;每个区块自己内部已经对齐
· LoginCard (含 input/button/link) · BrandHeader · LegalFooter 3 个区块,4 个文本调整
结果:视觉接近设计稿,需个别微调。 还原度 ~ 90%
C
大颗粒 · 1 模板
AI 决策点
~3 次选择
直接用 LoginTemplate,改 logo / 文案 / 主色
· LoginTemplate ↳ 改 brand text ↳ 改 hero image ↳ 改 footer copy 完。
结果:跟稿几乎完全一致,但缺灵活度。 还原度 ~ 99%
" 用 1000 个积木拼容易出错, 用 10 个积木大概率还原度高。 —— 设计师在汇川咨询里的洞察
02 / 选择空间

为什么
越多越歪?

AI 拼组件,每一步都要在所有可选项里挑一个。 颗粒度越细,步骤越多 —— 出错概率指数级膨胀。

03 / 上下文腐烂

越拼到后面,
AI 越健忘。

颗粒度小 = 步骤多 = 对话长。 AI 拼到第 5 个组件时,已经开始遗忘第 1 步的需求。 这就是"上下文腐烂"。

⭐ 互动点"下一步" 看 AI 拼一个组件 —— 左边的需求会随时间变灰、变划线(被遗忘)。

04 / 预制菜怎么来

复合组件的
3 种沉淀来源。

知道颗粒度要大,接下来:预制菜从哪儿来? 范米花儿在咨询里讲了三种方式 —— 每种适合不同场景。

01 · 来源 A

对话生成

跟 AI 自然描述:"帮我做一个页头组件,含面包屑、标题、右侧操作区"。AI 直接吐源码,你看完保存到 components/blocks/。

你: "做一个工作台数据卡片, 含标题、数字、对比箭头" AI: <StatCard /> 已生成, 放到 components/blocks/
·· 优 快 · 直接说人话
·· 劣 视觉细节难以精确

适合:基础区块(页头/侧栏/卡片)

02 · 来源 B

Figma 1:1 还原

把设计稿丢给 AI 走 Figma MCP,要求"用我已有的 base 组件 + token 还原"。AI 拼出来的复合组件视觉精确。

你: [扔 Figma frame 链接] "用 base + token 还原" AI: 读图层 → 调用 Button/Input/... <LoginCard /> 已生成
·· 优 视觉精确 · 跟稿一致
·· 劣 要 MCP · 要稿子先画好

适合:有完整设计稿的页面区块

03 · 来源 C

开源二次封装

复杂组件(表格 / DatePicker / 富文本)不要自己造。找开源的,套上你的 token 包一层。

你: 选 TanStack Table 作底层 AI: 包装成 <CompanyTable /> ↳ 加分页 / 筛选 / 排序 ↳ 用你的 token 改皮
·· 优 功能强 · 久经考验
·· 劣 要懂点工程选型

适合:复杂表格/表单/编辑器

提示 3 种是叠加的,不是选一个。一个真实项目里,基础区块用 A,关键页面用 B,复杂功能用 C。
05 / 泛化能力

库里没有的,
它能拼出来吗?

这是个高频问题:"我的复合组件库没穷尽所有场景,AI 遇到没见过的需求怎么办?"

YOUR LIBRARY 已有体系
"近邻"
变体 ✓ 小调整 ✓ 组合 ✓ 完全陌生风格 ✗ 特效卡片 ✗ 3D 装饰 ✗

能 · 但有边界。

AI 的"泛化"不是凭空创造,而是在你已有体系的"近邻"里插值 —— 变体 / 微调 / 组合,它都能搞定;但完全陌生的风格(C 端特效 / 3D 装饰 / 新拟物)它做不出来。

所以你的库越完整,AI 的"泛化能力"看起来越强。本质上 AI 在抄你 —— 抄你已经定的味道。

"我特别去尝试,让它生成一些库里没有的模块 —— 拼出来的内容也是看得过去的。 因为我这里面本身有这些内容,它会自学习去生成一个还可以的效果。" —— 范米花儿
关键 要做完全陌生的东西(比如 C 端营销特效)? AI 帮不了多少, 设计师手搓仍然是必经之路。

· 什么时候该升级颗粒度?

信号说明
AI 每次拼出来都走样 → 颗粒度太细。先做几个高频区块,放进 components/blocks/
同一个区块反复跟 AI 描述 → 该沉淀了。下次让 AI 调用,不要重新拼
5 个人做同种页面风格不一 → 升级到页面模板。每种页类型 1 个 starter
AI 总是遗忘最初的需求 → 上下文腐烂。把需求编进 rules 文件而不是聊天里
完全陌生的视觉(特效/3D) → AI 帮不了。设计师手搓,做完再沉淀进库
