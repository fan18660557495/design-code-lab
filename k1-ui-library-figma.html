01 / 一句话

一个抄进来，
一个借来用。

这俩名字听起来都是"UI 库"，但骨子里完全不同。 先把这层差异讲透，后面所有对比都从这里长出来。

SHADCN/UI · 抄进来

把代码抄进你家

下载一份开源 Figma kit，detach 之后
所有图层都能改。

用一个叫 npx shadcn add button 的命令，把 Button 组件的源代码直接复制到你项目的 components/ui/ 文件夹里。

之后这份代码就是你的了。改字号、改颜色、删一半、加新功能，随便。

→视觉风格：极简 · 黑白 · 大留白（欧美 SaaS 风）
→自由度：满分。源码就在眼前
→代价：组件少 · 复杂场景要自己拼
ANT DESIGN · 借来用

用公司的盒子

enable 公司发布的 team library，
组件只能用，进不去改图层。

用 npm install antd 把整个组件库装在角落（一个叫 node_modules 的隐藏文件夹），写代码时引用它的组件。

组件源码不归你——能调它给的 props（开关），但深度定制要走"主题配置"那一套，绕。

→视觉风格：蓝色为主 · 信息密 · 像中后台
→自由度：受限。深改容易翻车
→好处：组件超全 · 复杂场景拿来即用
" shadcn 是抄进来 —— 代码在你家，能改。 Antd 是借来用 —— 代码在它家，只能调。 —— 这一句话记住，后面所有差异都顺了
02 / 代码归谁

同一个 Button
住哪儿？

打开两个项目的文件夹长这样 —— 位置不同 = 你能不能改它。hover 任意一行看说明。

玩法绿色 ✏️ 你的 = 源码归你，随便改；灰色 🔒 锁住 = 在 node_modules 里，看得见但动不了。
your-project/ (shadcn) 抄进来
▸components/
▸ui/
⌥button.tsx✏️ 你的
⌥input.tsx✏️ 你的
⌥card.tsx✏️ 你的
⌥dialog.tsx✏️ 你的
⌥App.tsx✏️ 你的
▸node_modules/
📦@radix-ui/...🔒 锁住
📦clsx, cva ...🔒 锁住
结论：Button 长什么样、怎么动 —— 全在 button.tsx 里。改它 = 你说了算。
your-project/ (antd) 借来用
▸src/
⌥App.tsx✏️ 你的
↳写: import {'{'} Button {'}'} from 'antd'
▸node_modules/
📦antd/🔒 锁住
⌥Button/index.tsx🔒 锁住
⌥Button/style.less🔒 锁住
📦Input, Form, Table ...🔒 锁住
结论：Button 长什么样 —— 在 antd 自己家 (node_modules/antd/)。你只能调它的开关，不能进去改图层。
FIGMA 类比 shadcn = 在 Figma Community 下个 free kit，拖进自己文件 detach 之后，所有图层任你改。
Antd = 用公司发布的 team library，组件 instance 在你画布上，但只能改 variants —— 想进图层？得回到 library 文件去改。

06 / AI 协作

你画了稿，
AI 来实现。

同一份设计稿，扔给 Cursor / Claude / v0， 在 shadcn 项目和 Antd 项目里会得到不同结果。

·· 你的体验
在 SHADCN 项目里
在 ANTD 项目里
01AI 写出来第一眼
✓ 接近你的稿子 —— shadcn 默认就是极简风，AI 也是这套审美。
× 默认很 antd 味 —— 蓝色按钮、紧凑表格，跟你欧美风设计稿差很远。
02想调成你的设计
✓ AI 直接改组件源码。你点 inspect 看一眼能验收。
× AI 改完你常发现没生效 —— 因为要改的是主题配置，不是组件本身。
03跟前端沟通
✓ 你说"改这个按钮"，前端就改 button.tsx —— 说人话。
× 前端会说"得改 theme token"、"得改 ConfigProvider" —— 得学黑话。
04做复杂表格 / 表单
× AI 经常要拼组件 —— 因为 shadcn 没有现成的"高级表格"。
✓ AI 一句 <Table /> 搞定 —— 这是 Antd 的主场。
⚠ 重要提醒 不要"硬切"。 你公司是 Antd 项目，就用 Antd 跟 AI 协作 —— 逼 AI 写 shadcn 会让整个项目割裂，最后两套样式打架，更难收拾。 选型是在项目立项时决定的事，不是设计师能临时切换的。

· Figma 类比速查

代码概念Figma 类比 (你已经会了)
shadcn 抄源码 从 Community 下个 kit，detach 之后所有图层任改。
Antd 引用组件 用 team library，组件 instance 只能调 variants，进不去图层。
ConfigProvider 改 library 的 variables —— 一改全改。
改单个组件 className 改这一个 instance 的 override，只动它自己。
tokens / design tokens 就是 Figma Variables。下一篇 K2 会展开。
