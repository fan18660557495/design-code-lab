01 / 三块拼图

Elements 面板
就三块东西

F12 一开，看上去花花绿绿一大片 —— 别慌。Elements 面板永远只有三块：左边 DOM 树、右边 Styles、下面 盒模型。看清这三块，就赢一半了。

三块怎么读

从左到右、从上到下，每一块解决一个问题。点一下试试。

1
DOM TREE · 左边
这页是怎么搭的

嵌套的 HTML 标签树。每一块视觉都对应一个标签 —— <div> 是盒子、<button> 是按钮。点开折叠图标，像看 Figma 图层面板一样。

2
STYLES · 右边
这块用了什么 CSS

选中左边某个标签 → 右边自动列出它身上所有的样式规则。颜色、字号、padding、border-radius —— 网页上你看到的每个像素都来自这里某条规则。

3
BOX MODEL · 下边
间距和尺寸长这样

一张套娃图：margin / border / padding / content 四层。颜色是固定的 —— 橙、黄、绿、蓝。Figma 里一个数字代表 spacing，浏览器里要分清是 margin 还是 padding，差别很大。

认住这三块的位置永远不变。Chrome / Safari / Firefox / Edge —— 只是颜色和按钮位置略有不同，"DOM 树 + Styles + 盒模型" 这个三件套在所有浏览器里都长一样。学会一次，到哪都能用。
02 / DOM 树

点哪一块，
就框出哪一块

DOM 树跟 Figma 的图层面板是一回事 —— 都是嵌套结构。区别只有一个：网页里它是 HTML 标签，每个标签有名字（div / button / h2）。点 DOM 里的某一行，左边页面预览会自动框出对应的元素。

玩法点下面 DOM 树里的任意一行 —— 左边的登录卡片对应的那块就会被蓝色虚线框圈出来，同时这一行也会高亮成蓝底。这就是浏览器里"图层 ↔ 页面"的双向同步。
DOM ↔ 页面 · 双向同步 已选中 · .login-card
— 浏览器里看到的页面

设计师视角DOM 树 ≈ Figma 图层面板。缩进越深的元素，相当于在 Figma 里被嵌套在更里面的 frame。最顶层那个 <html> 就是 Figma 里的 page，往下逐层是 frame、组件、元素。

04 / 盒模型

一个盒子
四层套娃

DevTools 最下面那张套娃图叫盒模型 —— 任何一个 HTML 元素都长这样：从外到里 margin → border → padding → content。颜色是 Chrome 固定的，记住对应关系，以后看到立刻知道哪是哪。

互动下面是放大版的盒模型。拖动右边的 padding 滑块 —— 你会看到绿色的那一层在变厚 / 变薄，按钮被"撑大或缩小"。这就是 padding 在做的事：从内容往外推。

设计师常踩坑"这两个块为什么挨这么近？" —— 一定要分清是外间距 margin 还是内间距 padding。同样是"间隔 16px"，写成 margin 是跟外面别的元素的距离，写成 padding 是自己内部内容到边的距离，效果完全不同。
05 / 5 个手艺

设计师最该
学会的5 招

会读三块还不够。这 5 招是日常你打开任何网页都会用到的 —— 从最基础的"右键检查"到"在 Elements 里直接改 CSS 看效果"，每一招都能让你跟开发的对话变得更有效。配快捷键 + 截图。

01 · 入门第一招

右键 → 检查

right-click · inspect

看到任何感兴趣的元素 —— 按钮、文字、图标、间距 —— 右键 → 检查（Inspect）。DevTools 自动打开，DOM 树自动跳到那一行，Styles 面板自动列出它的样式。这一招你一辈子用到底。

Mac ⌘+⌥+I Win F12 Ctrl+Shift+I
— 右键菜单
▸ 复制
▸ 粘贴
▸ 翻译

02 · 改完立刻看

直接改 CSS
当场看效果

live-edit · in browser

在 Styles 面板里 —— 任何样式都能直接点进去改。改完按 Enter，浏览器立刻显示新效果。颜色不喜欢？改一下试试。padding 不够大？拖一拖。这是设计师跟开发对话的最强语言。

注意这里的修改只在你的浏览器里临时生效，刷新页面就没了。要永久生效，要让开发改源代码 —— 但你已经知道"改成什么样最好看"了，沟通就赢了一半。
— Styles 面板，点进去改
.button {
background: #ff4d2e;
padding: 12px 20px;
}
03 · 看真值

Computed 面板
看最终的样式

computed · final values

Styles 面板列的是"写了什么"。但同一个元素往往被好几条规则同时影响 —— 哪条赢了？最终生效的字号、间距、颜色到底是多少？切到 Computed 面板，浏览器算好了直接告诉你。设计师对真实数值有疑问，先看这。

移动端 + 深色模式 · 一键切换

device toolbar · responsive · dark mode

设计稿里的"移动端版本""深色模式版本"在浏览器里也是一键切换看效果。点 DevTools 左上角的"手机+电脑"图标 → 选 iPhone / iPad / 自定义尺寸。再到 Rendering 面板里，找 prefers-color-scheme，切到 dark —— 整页立刻进入深色模式。

PART 01 · 三块

Elements 三件套

3 zones to read
左 · DOM 树
HTML 嵌套结构 —— 跟 Figma 图层面板是一回事。每一行对应页面上一块视觉。
右 · Styles
选中元素的所有 CSS。能直接点进去改、立刻看效果。
下 · 盒模型
四层套娃图：margin / border / padding / content。看间距和尺寸全在这。
PART 02 · 颜色

盒模型四色

box model palette
橙 · MARGIN
外间距 —— 跟别人保持距离。会撑大占地面积。
黄 · BORDER
边框线 —— 实际画在屏幕上的那条线。
绿 · PADDING
内间距 —— 自己内部内容到边的距离。背景色会延伸到这里。
蓝 · CONTENT
内容本身。文字、图片、子元素就装在这里面。
