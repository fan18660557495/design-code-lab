Mask & Clip

设计稿里“只显示这一块”可能对应三种完全不同的代码：图片比例裁切、容器溢出隐藏、或者按形状遮罩。 这篇先把它们分清，再决定用 object-fit、overflow、clip-path 还是 mask。

目标可见区域
心智模型Viewport over content
CSS 属性object-fit / clip / mask
结论先判断裁谁
crop

裁图片比例

内容是图片，容器比例固定，优先想 object-fit。

clip-path

裁元素轮廓

元素整体要变成斜切、多边形、圆形时，用 clip-path 更直接。

mask

按透明度挖掉

需要渐隐、挖洞、票券缺口时，mask 更接近设计语言。

01 / 先拆设计语言

先问一句：
你在 裁谁？

裁图片、裁容器、裁可见像素是三件事。看起来都是“遮住一部分”，但对应的 CSS 模型不同。

WRONG MODEL

所有裁切都上 overflow: hidden

overflow 只能处理超出盒子的内容。它不能把元素本身变成斜切形，也不能做半透明渐隐。

.shape { overflow: hidden; /* 只能藏溢出，不能定义形状 */ }
RIGHT MODEL

按目标选择属性

图片比例用 object-fit；几何轮廓用 clip-path；透明度和挖洞用 mask。先分类，代码就会稳定很多。

.media { object-fit: cover; } .shape { clip-path: polygon(...); } .fade { mask-image: linear-gradient(...); }
01裁图片只是不想图片变形，先用 object-fit 和 object-position。
02裁溢出内容超出圆角容器，用 overflow hidden。
03裁轮廓元素要变成特殊几何形，用 clip-path。
04裁透明度渐隐、挖洞、复杂 alpha，用 mask。

03 / 设计到代码的对照

        
四个属性，
别 混着用。

      

      

        能用最简单的属性解决，就不要上最复杂的属性。复杂不是问题，混乱才是问题。
      

    


    

      
        
你要达到的效果首选属性注意点

      
      
        
图片填满固定比例卡片
不能被压扁		object-fit: cover检查主体是否被裁，必要时加 object-position。

        
内容超出圆角容器
不要溢出来		overflow: hidden会影响浮层、tooltip 和 sticky 子元素。

        
元素变成几何形
斜切、多边形、圆形		clip-path普通 border 和 box-shadow 可能不会按轮廓走。

        
渐隐或挖洞
按 alpha 控制可见性		mask-image注意浏览器前缀和组合模式差异。

      
    

  






  

    

      

        
04 / 交付口径

        
把裁切说成
可见区域规则。

      

      

        给前端或 AI 的描述要避免“这里裁一下”。应该说清楚裁的是图片内容、容器溢出、元素轮廓，还是透明度。
      

    


    

      

        01 / CONTENT
        
说明主体位置

        
封面图裁切要写 object-position，尤其是人物、产品和关键图标。

        
  • center top
  • left 40%
  • 保留主体完整

      

      

        02 / SHAPE
        
给出轮廓来源

        
如果是特殊形状，最好给 SVG path、polygon 点位或可复用变量。

        
  • clip-path polygon
  • SVG clipPath
  • mask gradient

      

      

        03 / HIT AREA
        
检查点击区域

        
视觉上被裁掉，不代表交互区域一定同步改变。异形按钮尤其要测。

        
  • hover 是否一致
  • focus ring 是否可见
  • 点击热区是否合理
