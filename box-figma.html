05 / 尺寸

Hug · Fill · Fixed
窗口变了，
谁来让步。

这三个词就是响应式的核心。拖下面的容器宽度看一眼 —— Hug 死死保持、Fill 主动让步、Fixed 撞了车也不让。

短文本

Hug = 紧贴内容（像气球贴皮肤） Fill = 撑满父容器剩余空间 Fixed = 给一个固定数字，不变

真实场景 · NAV BAR

3 种宽度，1 套规则。

同样的导航栏，桌面 / 平板 / 手机 三种宽度。规则只有一句： Logo 和 Avatar 是 Hug，中间空白是 Fill。 窗口缩小时，Fill 自动让步，其他保持不动 —— 这就是响应式。

08 / 实践

回到开头：
怎么让 AI 真的能用
你的设计稿？

你已经知道 Auto Layout 是什么了。下面 3 条建议，是把这件事变成肌肉记忆的关键。

i

所有"组"都用 Auto Layout

看到一组横排的标签、一个卡片、一个表单 row —— 框选 Shift+A。一个 Frame 套一个 Frame，让结构能被读出来，而不是 100 个绝对定位的图层堆在那。

ii

给图层起人话名字

Card / Header / SearchBar 比 Frame 247 强一万倍。AI 会把图层名当组件名，名字乱代码就乱。

iii

明确 Hug / Fill / Fixed

每个元素都问一句：它的宽度该怎么变？响应式问题 80% 出在这里没想清楚。AI 不会替你做这个决策，它只能照搬你定的策略。

划重点 · TL;DR

做不到 100% Auto Layout、图层名也来不及全改？没关系。把"主结构"做对就够了——卡片、列表、导航这种会被反复用到的块。剩下的乱一点也行。

真嫌麻烦，可以直接把稿子丢给 AI，告诉它："帮我把这份稿子的图层重新组织成 Auto Layout、起好名字。" 只要这一句话就够了。
