04 / 三种画法

同一个按钮 ·
AI 读到天差地别。

视觉上完全一样的紫色"继续"按钮 —— 三种画法在 JSON 里暴露的信息完全不同。 AI 拿到多少 context,直接决定还原度。

A · 推荐

主组件 + variant + variable

继续
{ "type": "INSTANCE", "name": "Button/primary", "mainComponent": { "id": "Btn:1:5" }, "componentProperties": { "Variant": "primary", "Size": "md" }, "boundVariables": { "fills": [{ "id": "--primary" }] } }
还原度 ~95%

AI 知道这是 Button 组件的 primary variant,绑了 --primary token。三层信息全有:类型 / 变体 / 主题绑定。

B · 凑合

主组件 instance + 硬色值

继续
{ "type": "INSTANCE", "name": "Button", "mainComponent": { "id": "Btn:1:5" }, // 没绑 variable, override 硬色值 "fills": [{ "type": "SOLID", "color": { "r": 0.36, "g": 0.28, "b": 0.98 } }] }
还原度 ~70%

AI 知道是 Button 组件,但不知道这个紫色叫 "primary" —— 它只看到 #5B47FB。主题切换时它接不住。

C · 翻车

自由画 · rectangle + text

继续
{ "type": "RECTANGLE", "name": "Rectangle 23", // 不是 INSTANCE ! "fills": [{ "color": { "r": 0.36, "g": 0.28, "b": 0.98 } }], "cornerRadius": 12, // 旁边浮一个独立 TEXT 节点 "继续" }
还原度 ~30%

AI 看到一个紫色矩形 + 一个文字 ——它不知道这是按钮。它要靠猜:"长得像按钮吧?"经常猜偏。

" 视觉一样,JSON 不一样,AI 拿到的上下文量天差地别。 "画法选择"就是"给 AI 的 context 选择"。
05 / 看不到的

AI 看到 / 看不到
的清单。

以下是 AI 通过 Figma MCP 真实能读 / 读不到的所有东西。 知道它的盲区,你就能给它补齐信息。

字段 / 信息 AI 看到? 备注
图层名字 ✓ 看到 "LoginCard" / "Button/primary" —— 你怎么命名它就怎么读。乱命名 = 自己挖坑。
尺寸 / 坐标 ✓ 看到 width / height / x / y 全有。AI 能精确还原布局。
颜色(硬色值) ✓ 看到 RGB 值。但不知道这色叫"主色"还是"警告色"。
Variable 引用 ✓ 看到 核心信号。boundVariables 字段告诉 AI "这是 --primary token,不是普通紫色"。
组件 instance + variant ✓ 看到 另一个核心。mainComponent + componentProperties 让 AI 知道"这是 Button 的 primary 变体"。
文本内容 + 字号字重 ✓ 看到 characters + style.fontSize 等。文本内容 1:1 拿到。
图标(SVG 形式) ⚠ 部分 SVG 路径 AI 读得到;但一堆图层堆出来的图标 AI 没法识别它是个 logo。
设计稿 comment 评论 ⚠ 部分 看 MCP server 实现。大多数情况读不到(不在 MCP 协议默认暴露范围)。
你脑里的 design intent ✗ 看不到 "这一块要给用户惊喜感" / "这是品牌一致性" —— 除非你写进 Dev Mode annotation。
隔壁页面的"风格语言" ✗ 看不到 AI 只读你指给它的那个 frame。不知道隔壁页面用了什么风格。
动效 / prototype 交互 ✗ 看不到 Figma prototype 的跳转、转场、interactive component —— 当前 MCP 几乎全部丢失。
你的"试错过程" ✗ 看不到 那 30 分钟你纠结的 5 个版本、回滚的、被砍掉的 —— AI 只看到最终结果。
01

用 main component

不要自由画。每个按钮 / 输入框 / 卡片都用 主组件 instance。AI 才能识别这是"按钮",而不是"紫色矩形"。

02

绑 variables,不写死

主色、间距、圆角全部用 Variables 绑。AI 能读到 boundVariables 字段,知道这是"主题色"而不是"某个 RGB 值"。

03

图层语义命名

"Submit Button" 好过 "Rectangle 23"。"LoginCard" 好过 "Frame 5"。AI 把 name 字段也当线索读。

04

写 annotation

Dev Mode 里可以加注释。把设计意图("这是用户首次登录的引导") 写进去 —— 这是 AI 唯一能读到的 intent。
