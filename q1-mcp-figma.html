MCP · Model Context Protocol
路线图 关于
Q1 · Design × AI 阅读 12 分钟 含 6 个场景剧场 不写代码也能装上

MCP 不是新 AI，
是 AI 的USB 接口。

MCP（Model Context Protocol）是让 AI 接外部工具的协议。
装上 Figma MCP AI 就能"看到"你的设计稿，
装上 Chrome MCP AI 就能"看到"你的真实网站。

本页讲什么MCP 是什么 / 怎么用
含拆解协议 + 角色 + 接入方式
面向用 Claude / Cursor 但没装过 MCP 的设计师
前置知识能跑 npm 即可
01 / 对比

没 MCP · AI 只能用嘴说

不装 MCP 的 AI 就像聋哑外宾 —— 你给它截图它能看， 但你的 Figma 文件、你网站的真实样子、你 GitHub 的 PR，AI 都摸不着。

没装 MCP · AI 的世界
你 · "帮我把这个 Figma 实现成代码"
→ AI 看不到你 Figma 文件
→ AI 不知道你网站长啥样
→ AI 不知道你 repo 里有哪些组件
→ AI 只能猜 / 让你截图喂它
AI · "请把设计稿截图发我..."
limit

AI 是个外宾

能听懂你说话、看你截图，但动不了手 —— 你的本地文件、Figma 内部、浏览器现场，它全够不着。

limit

每次都要人肉传话

"截个图给我看看" / "复制一下代码片段" / "查一下 commit 历史" —— AI 干不了，全靠你来回搬运。

limit

所以 AI 永远在瞎猜

它生成的代码、给的建议，都基于"你刚才说了什么"， 不是你真实项目的状态。错得多正常。

所以 MCP 就是为了让 AI 长出"手"—— 装一个 MCP server，AI 就能直接读 / 写对应的工具。 不再需要你做人肉传话。
02 / 协议

MCP 协议
4 个角色

MCP 不是某个软件，是 Anthropic 2024 年发布的通信协议。 理解它就 4 个角色：客户端 → 协议 → 服务端 → 外部工具。

step 1 · client
AI 工具
Claude Code · Cursor · Claude Desktop
→
step 2 · protocol
MCP 协议
JSON-RPC
over stdio / HTTP
→
step 3 · server
MCP Server
Figma 写的
Chrome 写的
GitHub 写的
→
step 4 · tool
外部工具
Figma · Chrome · GitHub · 文件系统
关键 AI 工具（client）只跟协议说话，不直接跟工具说话。 所以一份 MCP 协议能接 100 个 server / 100 种外部工具 —— 就像 USB 接口能接 100 种设备一样。
03 / 装法

远程 vs 本地
两种装法

每个 MCP server 都有两种部署模式 —— 远程（云端 / OAuth 一键 / 免费）vs 本地（装 desktop app / 功能更全）。 设计师 95% 的场景选远程就行。

推荐 · 远程

远程 Server

接入：OAuth 一键 (5 分钟)
价格：免费 · 所有 Figma 计划
功能：读 + 写 + Code Connect
适合：95% 设计师

# Claude Code
$ claude plugin install figma

# Cursor: 点 MCP Catalog 一键
高阶 · 本地

本地 Server (Desktop)

接入：装 Figma Desktop app + 手动配置
价格：Dev / Full seat 必须
功能：全部 + 写 canvas (新 features)
适合：每天 8 小时和 Figma 打交道

# 1. 开 Figma Desktop
# 2. Preferences → Enable MCP Server
# 3. http://localhost:3845 暴露给 client
先这样 第一次用直接选 远程 server 接 Claude Code 或 Cursor —— 5 分钟搞定。等你重度使用了再切本地。
04 / 场景

装上 MCP·
AI 解锁 6 件事

6 个真实场景 —— 你想干什么 → 装哪个 MCP → AI 解锁什么能力 → 实现你的需求。点任意一张卡片，看完一出 4 幕戏。

注意：MCP 不是给你直接调用的 —— 是给 AI agent 用的。 你装好 MCP 之后，AI 在背后自动调用对应工具， 你跟 AI 用自然语言说话，AI 自己决定怎么用 MCP。 下面 6 个场景演示装上对应 MCP 后，AI 能做什么。
领悟 MCP 才是 Agentic 的真正基石 —— 没 MCP 的 AI 只能"用嘴说"，装上 MCP 才能"动手"。 下一篇我们讲 Agent —— AI 怎么用这些工具规划、执行、反思整个任务。
