路线图 关于
Q1 · Design × AI 阅读 10 分钟 5 张可视图 + 装配台 不写代码也能装上

MCP 不是新 AI，
是 AI 的USB 接口。

MCP（Model Context Protocol）是让 AI 接外部工具的协议。
装上 Figma MCP AI 就能"看到"你的设计稿，
装上 Chrome MCP AI 就能"看到"你的真实网站。

本页讲什么MCP 是什么 / 怎么用
含拆解协议 + 角色 + 接入方式
面向用 Claude / Cursor 但没装过 MCP 的设计师
前置知识能跑 npm 即可
01 / 对比

没 MCP · AI 是聋哑外宾
装上 MCP · AI 长出手

左边：AI 只能用嘴说，外面世界全够不着。
右边：装上 MCP，AI 伸出触手，接 N 种外部工具。

no · mcp
没 MCP · AI 是聋哑外宾
isolated · no hands · no tools
你给截图 → AI 看 → 嘴上回答
Figma / Chrome / GitHub 全在外面 · 摸不着
with · mcp
装 MCP · AI 长出 N 只手
tentacles → figma / chrome / playwright / github / fs
装一个 MCP server · 长一只手
AI 能动手 · 读写真实工具 · 不再靠你传话
所以 MCP 就是 AI 的"手" —— 装一个 server，长一只手； 装 5 个 server，AI 同时摸到 Figma / Chrome / GitHub / 本地文件 / Playwright。
02 / 协议

MCP 协议
4 角色 · 数据包来回流

MCP 不是某个软件，是 Anthropic 2024 年发布的通信协议。 看 request 橙色包 → 怎么从 client 流到 tool，response 黑色包 ← 怎么流回来。

MCP Server
Figma / Chrome /
GitHub 等写的
tool
外部工具
真实在用的
Figma / 浏览器 / FS
关键 AI 工具（client）只跟协议说话，不直接跟工具说话。 所以一份 MCP 协议能接 100 个 server / 100 种外部工具 —— 就像 USB 接口能接 100 种设备一样。
03 / 装法

远程一朵云
本地一台 Mac · 两种装法

每个 MCP server 都有两种部署模式 —— 远程云端 OAuth 一键；本地装 desktop app + 手动配置。 设计师 95% 的场景直接选远程。

推荐 · remote
远程 Server · 云端
cloud · oauth · 5 min · free

免费 · 所有 Figma plan · 读 + 写 + Code Connect · 适合 95% 设计师 

# Claude Code
$ claude plugin install figma

# Cursor: 点 MCP Catalog 一键
高阶 · local
本地 Server (Desktop)
localhost:3845 · manual · all features

Dev / Full seat 必须 · 写 canvas 新 features · 适合每天泡 Figma 的人 

# 1. 开 Figma Desktop
# 2. Preferences → Enable MCP Server
# 3. http://localhost:3845 暴露给 client
先这样 第一次用直接选 远程 server 接 Claude Code 或 Cursor —— 5 分钟搞定。等你重度使用了再切本地。
