浅色稿改 Dark，
看着像是

BAD · 散装颜色硬切

反白一刷，问题全冒出来

底色直接换黑，文字直接换白。但所有"次要色"（图标背景、按钮背景、徽章背景）当初是按浅色背景配的。在深色里一下子全坏掉。

03 / 配合

Theme +
Context，
怎么配合？

前面 B1 学过 React Context（"App 级别共享一个值"）。Theme 就是一个最经典的 Context 用法： ThemeProvider 包整个 App，子组件用 useContext 读当前 theme，谁都不用通过 props 一层一层传。

PROVIDER ThemeProvider · theme="dark"
↓ theme 通过 Context 广播给整棵树
CONSUMER <Header />
CONSUMER <Sidebar />
CONSUMER <Main />
DEEP <NavItem />
DEEP <UserMenu />
DEEP <Card />
DEEP <CTA />

① 顶层放 Provider

包住整个 App。Provider 内部把 theme 写到 html[data-theme]，CSS Variables 立刻生效。

function App() {
  const [theme, setTheme] = useState('light');
  return (
    <ThemeProvider theme={theme}
      setTheme={setTheme}>
      <Layout />
    </ThemeProvider>
  );
}

② 子组件就近读

任何深度的子组件，useTheme() 一行就拿到当前主题。不用一层层 props 传。

function ThemeToggle() {
  const { theme, setTheme } = useTheme();
  return (
    <button onClick={() => setTheme(
      theme === 'light' ? 'dark' : 'light'
    )}>切换</button>
  );
}

第三层 Provider 给一个特殊 CTA 打上品牌紫。它不知道外面是 Light 还是 Dark —— 它只读"最近那一层"。

观察 · NOTE

切到 Brand 时，橙色品牌色变成了金黄 —— 不是因为我们重写了组件，而是 --m-primary 这个 token 在 Brand 主题下的值就是金黄。组件代码一行没改。

06 / 速查

怎么真的把
Theme 系统
跑起来？

三个阶段 —— 设计、命名、覆盖。每一步都有"工程师/设计师都能照做"的清单。

01

Tokens 设计阶段

  • 先梳理"语义层"：bg / surface / text / primary —— 不是颜色名
  • 每个语义 token 都要在每个 Mode 里有值，缺一个就翻车
  • 不要在组件里用色码，全用 var(--*)
  • 截屏每个 Mode 验对比度，WCAG AA 是底线
02

Mode 命名规则

  • 三个主题就用 light / dark / brand，别用 1 / 2 / 3
  • Figma Mode 名 = CSS data-theme 值，一字不差
  • Variable 名用 "/" 分层（color/text/soft），代码侧用 "-"（--color-text-soft）
  • 新增主题 = 新增一个 Mode + 对应 CSS 块。组件不改
03

局部覆盖

  • 大部分场景不需要嵌套 Provider，一层就够
  • 真要嵌套：营销 Banner / 商品详情主图区 / 错误页这种"局部品牌时刻"
  • 嵌套时把 data-theme 写在子节点的 div 上，不是 html 上
  • 测试时三层都点一遍，确认每层切换互不干扰

设计师 · 交付清单

把这份给前端，他/她不用回头问你任何颜色相关的事。

  • Variables 表：所有 color / spacing / radius，每个都有 Light/Dark/Brand 三栏值
  • Mode 名：在 Figma 里就叫 light / dark / brand
  • 截屏：每个 Mode 一张全页截图，前端对照检查
  • 边界说明：哪些组件"不跟主题变"（比如 Logo）特别标出

工程师 · 实施清单

一份 theme.css + 一个 ThemeProvider，覆盖 90% 场景。

  • theme.css：三个 html[data-theme] 块，token 值一一对照
  • ThemeProvider：用 Context 暴露 theme + setTheme，把值同步到 html
  • localStorage：记住用户选择，刷新不重置
  • 系统偏好：默认读 prefers-color-scheme，跟随系统更友好
