01 / 拆开

Skills 就是
一个文件夹

你以为是某种 AI 黑魔法，其实就是个文件夹 —— 里面有一份 markdown 当入口， 其它都是可选的脚本和参考资料。点点看里面装了什么。

默认状态

左侧点一个文件 / 文件夹

只有 SKILL.md 是必须的。 其它三个文件夹（scripts / references / templates）都可选 —— 你想做什么 skill，就装什么进来。 Claude 启动时只读 SKILL.md 的最前几行；正文和这些附件，用到才打开。 

# 最小可用的 skill 长这样：
pptx/
└── SKILL.md           # 50 行 markdown，搞定
类比 想象一本食谱书：SKILL.md 是封面 + 目录（必须）； references/ 是后面厚厚的术语表（查的时候才翻）； scripts/ 是配套的电动料理机（要的话拿来用）； templates/ 是空白食谱卡（按场景填）。 Claude 启动时只看封面 —— 用到了才往里翻。
02 / 剖开

SKILL.md 内部
5 个秘密

点开 SKILL.md，第一眼你会看到一段奇怪的 --- 包起来的 YAML， 然后是 markdown 正文。这 5 个位置每个都有讲究 —— 写错任何一个，Claude 就不用你的 skill。

SKILL.md 
---
name: pptx
description: Use this skill any time a .pptx file is involved — creating decks, reading slides, extracting speaker notes...
allowed-tools: [Read, Write, Bash]
---
 
# PowerPoint Skill
 
## When to use
用户提到 .pptx 文件、deck、slides、presentation 时用。
 
## Steps
1. 用 extract_text.py 抽出文字。
2. 拼底层结构 — 参考 @references/pptx_format.md
3. 不确定就读 @references/pptx_format.md
秘密 · 01

---YAML frontmatter

最上面三道横杠包起来的就是 frontmatter。Claude 启动时只看这一段，剩下的暂时不读。name + description 是必须的，其它全选填。

秘密 · 02

description就是触发器

这一行决定了 Claude 看到用户消息时会不会想到你这个 skill。写得糟 → 看不见 → 不触发。这是 skill 最容易写崩的地方 —— §04 会专门玩这个。

秘密 · 03

allowed-tools是栅栏

限制这个 skill 启动后只能用哪些工具。比如一个 "只读" 的 skill 就不应该有 Write 权限。不写这行 = 默认全开。安全和能力边界，靠这一行画。

秘密 · 04

@references/xxx按需加载

正文里写 @references/pptx_format.md，Claude 读到这一行才会去打开那个文件。所以 skill 的"知识库"可以塞得很大 —— 不用的时候不烧 token。

秘密 · 05

正文就是 markdown

没有 DSL，没有 JSON，没有 schema。就是你写 Notion 那种 markdown。标题分 When to use / Steps / Tips 几块，Claude 当文档读 —— 你能看懂的，它也能。

最关键 整份 SKILL.md 里，description 是杠杆最长的一行。 其它都可以慢慢补，只有 description 决定了"Claude 到底会不会想起来用你写的这个 skill"。 写得糟 = skill 写完不会被触发 = 等于没写。
03 / 加载

三层金字塔
用到才读

100 个 skill 同时挂在 Claude 上，context 会不会炸？答案是不会 —— 因为 skill 走的是 progressive disclosure（渐进披露）：先看一行简介，匹配上了才往里翻。

› user:点上面任一个，看 Claude 怎么往下挖
› idle · 等用户输入
类比 Tier 1 = 菜单封面（一眼扫，决定要不要进店）· Tier 2 = 菜单内页（进店了再翻菜的做法）· Tier 3 = 后厨配方（点了某道菜厨师才去翻）。 整个机制最妙的地方：100 个 skill 挂着，启动只读 100 行。
04 / 速查

三张
速查卡

全文压缩到三张卡。下次写 skill 之前先翻一遍 —— 印出来贴显示器边上也行。

CARD 01 · 结构

文件夹长这样

只有第一项必须
my-skill/
├── SKILL.md          # 必须
├── scripts/          # 可选 · 执行脚本
│   └── do_thing.py
├── references/       # 可选 · 长资料
│   └── rules.md
└── templates/        # 可选 · 模板素材
    └── starter.txt
记住
最小可用 skill = 一个文件夹 + 一份 SKILL.md。其余都是后来加的。
CARD 02 · 公式

SKILL.md 公式

YAML + markdown
---
name: my-skill          # 必须
description: ...        # 必须 + 关键
allowed-tools: [...]    # 可选
---

# When to use
# Steps
# 参考 @references/xx.md
description 公式
动词开头 + 具体场景清单 + 3-5 个 sample queries + DO NOT 排除清单。
CARD 03 · 流程

触发的全过程

用户问 → Claude 想 → skill 用
TIER 1用户说话 · Claude 把消息和所有 skill 的 description 对一遍（只读这一段）。
TIER 2命中了 · 打开匹配 skill 的 SKILL.md 正文，按 When / Steps 走。
TIER 3读到 @reference · 才真的打开那个文件 —— 不读不烧 token。
DONE跑完返回 · 离开 skill，回到普通对话，下一轮重来。
记住
Tier 1 = 入门券，没这关后面都没机会。所以 description 是杠杆。
