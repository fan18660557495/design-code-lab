01 / 问题

18 个 Variants
= 几个组件？

设计师做了一个按钮，加了 3 个 Variant、3 个 Size、2 个 Disabled —— 共 18 个变体。 前端打开代码看完，说："就一个组件，3 个 prop。"

设计师视角

"我做了 18 个变体"

在 Figma Variants 矩阵里看见 18 格，每格都是一个独立的 frame。心里默念：18 个组件 = 18 件事。

前端视角

"这是 1 个组件 + 3 个 prop"

代码里只有一个函数。3 个参数的排列组合 = 18 种长相。改一个共用样式只改一处。

function Button({ variant, size, disabled }) { /* ... */ }
02 / 类型

Property 有
4 种类型，
按钮的 4 个轴。

Figma 加 Property 时让你选类型 —— 4 种全部用上了。 我们的按钮恰好用了 4 个轴，每个轴对应一种类型。

            function Button({
              variant = "primary",
              size = "md",
              icon,
              disabled,
              children
            }) {
              return (
                <button
                  className={`btn ${variant} ${size}`}
                  disabled={disabled}
                >
                  {icon && <Icon />}
                  {children}
                </button>
              );
            }

        设计师在 Figma 选的值，最终决定屏幕上你看到的颜色 / 大小 / 图标。
        中间经过的"调用代码"、"组件参数"、"className"、"CSS 规则" —— 都是 React 代码里实实在在的"中转站"。
