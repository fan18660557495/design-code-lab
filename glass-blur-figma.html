毛玻璃 / 背景模糊
Backdrop Blur

毛玻璃不是把卡片本身 blur 掉，而是让卡片背后的内容被模糊、降噪、再透出来。 代码里关键不是一个 blur 值，而是 backdrop-filter、半透明底色、边缘线和降级方案的组合。

目标透 / 模糊 / 可读
心智模型Frosted overlay
CSS 属性backdrop-filter
结论模糊前先做底色
balanced

能看见背景

半透明底色 + 适度模糊，背景还在，但不会抢文字。

too opaque

像普通卡片

底色太实，玻璃感会消失，只剩一张浅色面板。

missing base

只剩噪音

透明度太低时，内容会被背景纹理和色块干扰。

01 / 先拆设计语言

它模糊的不是自己，
是 背后。

Figma 里的 Background Blur 对应 CSS 的 backdrop-filter。它和 filter: blur 最大的区别是：一个处理背后，一个处理自己。

WRONG MODEL

用 filter: blur() 模糊卡片

filter 会让卡片和里面的文字一起糊掉。它适合处理图片本身，不适合做清晰文字上的玻璃面板。

.glass { filter: blur(16px); /* 文字也会一起糊 */ }
RIGHT MODEL

用 backdrop-filter 处理背景

面板自己保持清晰，只把后面的画面过滤。再配合半透明底色和边缘线，玻璃才稳定。

.glass { background: rgba(250,247,240,.46); backdrop-filter: blur(16px); }
01先铺底色没有半透明底色，blur 后的背景仍然会干扰文字。
02再加模糊backdrop-filter 作用在背后，不是让内容糊掉。
03补边缘线浅色内描边和高光能让面板从复杂背景里分出来。
04准备 fallback不支持 backdrop-filter 时，退回更实的背景色。
02 / 样式拆解台

玻璃感，
靠 四个参数。

试着调 blur、底色透明度、饱和度和边缘线。你会看到：blur 越大不一定越好，透明度太低会直接伤害可读性。

03 / 设计到代码的对照

        
三个坑，
最容易 翻错。

      

      

        毛玻璃效果看起来很视觉，但实现上有明确边界：容器背景、父级层叠、浏览器支持都会影响结果。
      

    


    

      
        
设计稿看到常见误判正确处理

      
      
        
Figma Background Blur
面板内部文字清晰		写成 filter: blur()用 backdrop-filter，并保持内容层不被 blur。

        
玻璃面板很透
背景色彩很丰富		只调 blur，不加底色先加 rgba() 底色，再决定 blur 大小。

        
边缘有轻高光
玻璃边界清楚		忽略 border 和 inset highlight加半透明 border、内阴影或 1px 高光。

      
    

  






  

    

      

        
04 / 交付口径

        
毛玻璃要交付
可读性条件。

      

      

        不要只标“背景模糊 16”。更实用的交付是：在哪些背景上用、文字颜色怎么保证对比、降级方案是什么。
      

    


    

      

        01 / CONTRAST
        
先看文字

        
背景越花，底色越不能太透明。玻璃感必须服从可读性。

        
  • 正文不要贴复杂图案
  • 底色 alpha 留安全值
  • 必要时加遮罩层

      

      

        02 / SUPPORT
        
准备降级

        
浏览器不支持时，退成更实的半透明面板，而不是让内容裸奔。

        
  • @supports 包一层
  • fallback 背景更不透明
  • 避免关键功能依赖玻璃

      

      

        03 / COST
        
控制范围

        
backdrop-filter 有渲染成本。小面积浮层可以用，整页大面积要谨慎。

        
  • 少用在滚动大容器
  • 减少叠加层
  • 移动端重点检查
