01 / 先拆设计语言

它模糊的不是自己，
是 背后。

Figma 里的 Background Blur 对应 CSS 的 backdrop-filter。和 filter: blur 的根本区别是：一个处理背后，一个处理自己。

WRONG MODEL

用 filter: blur() 模糊卡片

filter 会让卡片和里面的文字一起糊掉。它处理图片本身可以，做清晰文字上的玻璃面板不行。

.glass { filter: blur(16px); /* 文字也会一起糊 */ }
RIGHT MODEL

用 backdrop-filter 处理背景

面板自己保持清晰，只把后面的画面过滤。再配合半透明底色和边缘线，玻璃才稳定。

.glass { background: rgba(250,247,240,.46); backdrop-filter: blur(16px); }
01先铺底色没有半透明底色，blur 之后的背景仍然会干扰文字。
02再加模糊backdrop-filter 作用在背后，不是让内容糊掉。
03补边缘线浅色内描边和高光让面板从复杂背景里分出来。
04准备 fallback不支持 backdrop-filter 时，退回更实的背景色。

毛玻璃的本质是
透过去看后面

当我们说一个面板"是玻璃的"，意思是：它允许背景被看见，但又把背景柔化到不抢内容的程度。

在浏览器里，这个能力由 backdrop-filter 提供。它不影响元素自己，只处理元素背后那一块画面。和 filter 完全不同。

所以做毛玻璃面板时，元素本身的文字要清晰，背景里的内容要被模糊和降噪——两者由两个不同的属性负责。

三个坑，
最容易 翻错。

      

      

        毛玻璃效果看起来视觉化，实现上有明确边界：容器背景、父级层叠、浏览器支持都会影响结果。
      

    


    

      
        
设计稿看到常见误判正确处理

      
      
        
Figma Background Blur
面板内部文字清晰		写成 filter: blur()用 backdrop-filter，并保持内容层不被 blur。

        
玻璃面板很透
背景色彩很丰富		只调 blur，不加底色先加 rgba() 底色，再决定 blur 大小。

        
边缘有轻高光
玻璃边界清楚		忽略 border 和 inset highlight加半透明 border、内阴影或 1px 高光。

        
父级有 transform / overflow
backdrop 不生效		以为属性写错了检查父级层叠上下文。某些 transform / will-change 会切断 backdrop。

      
    

  






  

    

      

        
04 / 交付口径

        
毛玻璃要交付
可读性条件。

      

      

        不要只标"背景模糊 16"。更实用的交付是：在什么背景上用、文字怎么保证对比、降级方案是什么。
      

    


    

      

        01 / CONTRAST
        
先看文字

        
背景越花，底色越不能太透明。玻璃感必须服从可读性。

        
  • 正文不贴复杂图案
  • 底色 alpha 留安全值
  • 必要时加遮罩层

      

      

        02 / SUPPORT
        
准备降级

        
浏览器不支持时，退成更实的半透明面板，而不是让内容裸奔。

        
  • @supports 包一层
  • fallback 背景更不透明
  • 关键功能不依赖玻璃

      

      

        03 / COST
        
控制范围

        
backdrop-filter 有渲染成本。小面积浮层可以用，整页大面积要谨慎。

        
  • 少用在滚动大容器
  • 减少叠加层
  • 移动端重点检查
