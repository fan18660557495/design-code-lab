01 / 问题

一个页面用了
7 个相似的红、
5 个相似的字号。

没系统的代价：前端凭感觉挑色，跟设计稿差半度；改稿一次，全站对齐失败。 Tokens 就是让"挑色"和"猜字号"这件事消失。

BAD · 散装值

每次都现挑一个

设计稿写 #FF4D2E，前端写 #ff5530，AI 看图猜了个 #f64a2c —— 同一个"品牌红"被实现了 7 次。改色等于改 7 个地方。

#FF4D2E #FF5530 #F64A2C #FF5040 #FC4928 #FF5235 #FA4530
GOOD · Tokens

一个名字，一个值

设计师定一个 color/brand/primary，前端写 var(--color-brand-primary)。改一次值，全站同步。这就是 Tokens 的全部意义。

--color-brand-primary --color-brand-primary --color-brand-primary

03 / 字号

字号不是随便挑的，
是一道阶梯。

Figma 里叫 Text Styles，代码里叫 --font-size-*。 阶梯的关键是比例—— 选一个比例系数（1.2、1.25、1.333…），整套字号自动算出来。

选一个比例

每一档的字号 = 上一档 × 比例。比例越大，字号差距越夸张；越小，整体越统一。试试切换：

/* type scale 自动算公式 */ :root { --font-base: 16px; --type-ratio: 1.25; /* 每一档 = base × ratiostep */ --font-size-display: 39.06px; --font-size-h1: 31.25px; --font-size-h2: 25.00px; --font-size-h3: 20.00px; --font-size-body: 16.00px; --font-size-small: 12.80px; --font-size-caption: 10.24px; }
为什么是比例？因为视觉上有"舒服的层级感" —— 类似音乐里的和弦。 1.25（大三度）、1.333（纯四度）、1.618（黄金比）都是经典选项。 选定一个比例，就别再瞎调中间值。

调整 Token 值

这两个滑块改的是 --space-4 和 --space-3 的值。注意整个卡片的呼吸感同步变化。

--space-4 · 卡片 padding 16px
--space-3 · 头像与名字间距 12px
--space-2 · 按钮之间 8px
/* 同步生成 */ .card { padding: var(--space-4); /* 16px */ gap: var(--space-3); /* 12px */ } .card-actions { gap: var(--space-2); /* 8px */ }
为什么固定 4 / 8 系统？因为屏幕的物理像素就是 4 / 8 的倍数最锐利， 而且数字少了选择困难症 —— --space-3 和 --space-4 二选一，比"13px 还是 14px"轻松一万倍。

06 / 实践

回到工作流：
从今天起，用名字，
不要用值。

Tokens 不是新鲜玩意，但它是后面所有视觉系统的地基。三条规矩记住，剩下的就是熟练度问题。

i

用 Semantic，别用 Literal

命名是 --color-brand-primary，不是 --color-orange。前者你换主题色不用改名字，后者就是个笑话。

ii

分组比扁平更清晰

用 color/text/primary 这种斜杠分组，到代码里变 --color-text-primary。用途 → 强弱，两层就够。

iii

能用 token 就别用值

Figma 里点菱形图标接 token，前端写 var(--*) 而不是 #ff4d2e。一个团队能不能换主题，全看这一条。

划重点 · TL;DR

Figma Variables 和 CSS Variables 是同一件事的两种界面——前者用面板点，后者用 var(--*) 写。命名规则定下来，整个团队就能对得上。

设计师常踩的坑：色板做了 30 个但代码里只用了 5 个。多余的会让前端混乱、AI 抓错色。 留住你真用上的那一组就够了。
