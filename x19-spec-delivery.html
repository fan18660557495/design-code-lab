01 / 问题

为什么稿子
开发完后总要返工 5 次？

因为你交的是视觉——但代码不止视觉。代码还要处理交互、数据、异常。 没说的部分，工程师只能猜——猜错了就返工。

稿子思维 · MOCKUP

"稿子 + 标注 = 完成"

设计师交一个 Figma 链接 + 标注：颜色、字号、spacing 都标全。看起来很完整。

工程师做完测试发现：按钮 disabled 颜色没标？empty state 谁画？接口报错怎么显示？提交超时呢？返工 5 次。

你：呃……每次都是临时补一个状态？

说明书思维 · SPEC

"规则说明书"

视觉 + 交互流程 + 数据约束 + 验收清单 四层一份文档。每层都写清，每层都可读。

工程师拿到：1 次到位写完代码，QA 照单写测试用例，PM 照单验收。三方对齐。

结果：返工 → 0 次。

幕二 · 拆开
03 / 拆开

规则说明书
解剖。

一份完整的说明书包含 5 个标准模块。每个模块都有 checklist—— 每张卡片展开看里面写什么、勾选"本模块就绪"。

1
描述
OVERVIEW
包含内容
  • 这是个什么组件 / 页面 / 功能
  • 目标用户 + 主要使用场景
  • 业务价值（为啥要做）
示例
  • 「确认删除 Modal」用在删除高风险数据前的二次确认
  • 用户 = 普通操作员；场景 = 列表勾选 → 删除 → 弹出
本模块就绪
2
视觉
VISUALS
包含内容
  • Figma 链接 + 标注（spacing / color / font / 8px 网格）
  • 所有状态截图：default / hover / active / disabled / loading / error / empty
  • 响应式 breakpoints（mobile / tablet / desktop）
常被漏
  • loading spinner 大小 / disabled 色值 / focus 描边
本模块就绪
3
行为
BEHAVIOR
包含内容
  • 交互流程（状态机 / 用户事件 → 系统反馈）
  • 异常处理：超时 / 冲突 / 双击 / 权限不足
  • 副作用：发邮件 / 写日志 / 刷新缓存 / 跳页面
示例
  • draft → submitting → success | fail
  • 双击提交按钮 = 仅触发一次
本模块就绪
4
数据
DATA
包含内容
  • 字段 schema：name · type · required · default · validation · computed · source
  • API 契约：endpoint / request / response / error codes
  • 状态住址：URL 参数 / 本页 state / 服务端 / 浏览器缓存
常被漏
  • 空值显示什么 / 错误返回什么 code / 字段长度上限
本模块就绪
5
验收
ACCEPTANCE
包含内容
  • 用户场景：Given / When / Then 三段式
  • 边界场景：empty / max / unauthorized / network fail
  • 性能要求：首屏 < N 秒 / 交互响应 < N ms
作用
  • QA 拿到这层 → 直接写测试用例
  • PM 拿到这层 → 直接照单验收
本模块就绪
说明书完整度
0 / 5 模块就绪
5 模块就是 工程 / QA / 产品三方共同合同 的写法。
谁拿到哪一层都能动手——这才叫交付。
勾选已就绪的模块 →

04 / 再做

轮到你——
给 用户注册流程

骨架已有，让你填"数据契约"和"验收清单"两个最难的层。 提交后会给你打分——不是看填了多少，看你填得对不对。

05 / 把这件事变成习惯

下次交付，
不发稿子链接。

从这一篇开始，把"发 Figma" 换成"过一遍 4 层 + 5 模块"。 工程师 / QA 会感谢你——返工 0 次的稿，第一次出现。

i

稿子下面附 4 段

视觉 · 交互 · 数据 · 验收。哪怕一句话——也比没强。

4 段都附，工程师 0 追问。

ii

验收先写 Given/When/Then

不写场景的设计 = 不可测。可测 = 可验收。

先写场景，再画稿——稿子反而更准。

iii

交付不是 send Figma

把稿子发出去后，跟工程师过一遍每个模块。对不上的地方现场补——比事后撕扯快 10 倍。

是确认对齐，不是丢链接。

划重点 · TL;DR

稿子是 视觉 × 一个组件 的产物——只占交付的 1/4。剩下的 3/4（交互、数据、验收）不写，就只能由工程师猜、QA 漏、PM 撕。

下次交付，附一张 5 项的 checklist——勾完才发。工程师 / QA 都会感谢你。
