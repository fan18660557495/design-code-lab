内翻圆角
Inverted Border Radius

一个角"凹进去"的卡片，在 CSS 里没有单一属性能做出来。最容易踩坑的是：以为有 negative border-radius；最容易出错的是：背景一换就穿帮。这篇把 4 种实现并排放，让你直观看到它们在不同背景下谁活着、谁穿帮。

目标异形外轮廓
心智模型把一个角挖掉
CSS 属性mask / clip / corner-shape
结论先看背景，再选方法
method · shadow

盖一块背景色

纯色背景上最便宜的做法。换背景立刻穿帮。

method · mask

真的挖掉像素

渐变、图片、毛玻璃背景都稳。检查边框和点击区域。

method · native

未来更自然

corner-shape 直接表达"这个角是 scoop"。需要 fallback。

01 / 先拆设计语言

它不是
负圆角。

设计稿里"圆角凹进来"，在 CSS 里其实是"轮廓被挖了一块"。没有 negative border-radius；最常用的 4 种实现各有取舍。

WRONG MODEL

找一个"负圆角"属性

border-radius 只能往内收，做不出凹进去的形状。也不要靠 transform / margin 拼接，会破坏盒模型。

.card { border-radius: -32px; /* 不存在 */ }
RIGHT MODEL

把"凹角"当成轮廓变形

关键是把元素的可见区域改掉：盖一块、挖一块、裁一块、声明角形。先确认背景，再决定方法。

.card { -webkit-mask: radial-gradient( circle 32px at 100% 0, transparent 98%, #000 101% ); }
01shadow伪元素 + 背景色盖角。只在纯色背景上稳。
02maskradial-gradient 挖透明洞。复杂背景也稳。
03clip-path路径裁切。形状最准，但 border/shadow 会被裁掉。
04nativecorner-shape: scoop。语义直接，需要 fallback。

03 / 四种方法对照

        
不是谁高级，
是 谁适合。

      

      

        同一个视觉，背后是 4 种不同的代码模型。不同实现对背景、边框、阴影和点击区域的影响完全不同。
      

    


    

      
        
方法原理背景支持限制

      
      
        

          shadow
伪元素 + 背景色盖角		
          用 ::after 画一个圆，颜色匹配父级背景色，盖在角落上。
          仅纯色。
          渐变 / 图片 / 玻璃背景立刻穿帮。
        

        

          mask
radial-gradient 挖洞		
          用透明 radial-gradient 把角落像素隐藏掉。
          渐变 / 图片 / 玻璃都稳。
          会同时裁掉 border 和 box-shadow 可见区域。
        

        

          clip-path
路径裁切		
          用 path() 或 polygon() 描出整个轮廓。
          所有背景都稳，形状最精确。
          需要按尺寸生成路径，或用 SVG clipPath。
        

        

          corner-shape
原生 scoop		
          声明 corner-shape: scoop，让边角天然内凹。
          支持时所有背景都稳，且 border / shadow 自然跟随。
          当前浏览器支持有限，需要 fallback。
        

      
    

  






  

    

      

        
04 / 交付口径

        
把"凹角"
说成 轮廓规则。

      

      

        给前端或 AI 的描述不要只说"右上角缺一个圆"。说清楚切口尺寸、所在背景、边框和阴影怎么处理。
      

    


    

      

        01 / CONTEXT
        
说明背景

        
"这张卡片在纯色 / 渐变 / 图片 / 毛玻璃背景上"。背景决定了能选 shadow 还是必须 mask / clip。

        
  • solid → shadow 也可以
  • 渐变以上 → mask / clip
  • 异形容器 → 优先 clip-path

      

      

        02 / SIZE
        
给出切口尺寸

        
切口直径 / 半径要标出来，最好用 token，不要每次重新拍数字。

        
  • --radius-cut: 32px
  • 注明哪个角
  • 大切口要测移动端

      

      

        03 / EDGE
        
说清边框 / 阴影

        
mask / clip 会同时影响 border / box-shadow。需要描边或阴影时，常常改用 filter: drop-shadow + SVG。

        
  • 有 border → 选 native / clip
  • 有 box-shadow → drop-shadow
  • 异形点击区 → 看 hit area
