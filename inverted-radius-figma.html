内翻圆角
Inverted Border Radius

设计稿里一个凹进去的圆角，前端不是写一个负数就结束。 这篇只拆一个效果：它可以用几种 CSS 方式做，每种方式适合什么背景，会牺牲什么。

目标凹角 / 切口
心智模型Box minus circle
可调参数半径 / 角落 / 背景
结论按场景选方案
method 01

纯色背景

用伪元素遮住角落，代码最轻，但背景一复杂就容易露馅。

method 02

复杂背景

用 mask 真正挖掉像素，图片、渐变、毛玻璃都更稳。

future

原生角形

未来的 corner-shape 会让边框和阴影跟着角形走。

01 / 先拆设计语言

它不是
负圆角。

普通圆角是把盒子外轮廓变圆。内翻圆角不是把半径写成负数，而是把一个圆从盒子角落挖出去。

WRONG MODEL

CSS 没有 border-radius: -24px

这类写法不会得到凹角。`border-radius` 描述的是盒子自己的凸角，规范里没有“向内凹”的负圆角关键字。

.card { border-radius: -24px; /* 无效：不是 CSS 的圆角模型 */ }
RIGHT MODEL

把一个圆从盒子里减掉

前端会把它翻译成“裁剪、遮罩、伪元素覆盖，或者未来的角形 API”。不是一个属性，而是一组选择。

.card { --cut: 32px; mask: radial-gradient(...); }

03 / 四种实现方式

        
不是谁高级，
是 谁适合。

      

      

        这个效果最容易误判的地方是：看起来都是一个凹角，但不同实现对背景、边框、阴影和点击区域的影响完全不同。
      

    


    

      

        
01 / SHADOW

        
伪元素 / box-shadow

        
最轻量，适合背景颜色固定的地方。它本质上是“盖住”角落。

        
    
          
  • 优点：兼容好，代码短。
    • 
          
  • 限制：复杂背景会穿帮。
    • 
          
  • 适合：后台卡片、票券、纯色面板。
    • 
        

      

      

        
02 / MASK

        
radial-gradient mask

        
用透明区域隐藏像素，更像真的把角挖掉。

        
    
          
  • 优点：渐变、图片、毛玻璃都更稳。
    • 
          
  • 限制：边框和点击区域要检查。
    • 
          
  • 适合：视觉稿背景比较复杂时。
    • 
        

      

      

        
03 / CLIP

        
clip-path / SVG clipPath

        
适合更复杂的轮廓，不只是圆形切口。

        
    
          
  • 优点：形状控制更精确。
    • 
          
  • 限制：普通 border / shadow 会被一起裁。
    • 
          
  • 适合：波浪、票券、异形卡片。
    • 
        

      

      

        
04 / NATIVE

        
corner-shape

        
未来更干净的原生写法，目标是让边框和阴影自然跟随角形。

        
    
          
  • 优点：语义直接。
    • 
          
  • 限制：目前仍需要看浏览器支持。
    • 
          
  • 适合：渐进增强，不做唯一方案。
    • 
        

      

    

  






  

    

      

        
04 / 复杂凹边

        
不是修圆角，
是 重画轮廓。

      

      

        如果缺口两端还要柔和过渡，单纯的 radial mask 就不够了。那已经不是“挖一个圆”，而是把整张卡片的外形重新画出来。
