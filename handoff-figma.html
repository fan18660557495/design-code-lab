§01 · PROBLEM

交付不到位的代价

设计稿一交出去，"是什么样"就不再由你控制。Handoff 决定了你的设计能在产品里还原到几分。

BAD · 散装交付

"我画完了，剩下就交给前端吧"

截图发到群里，没有 Variables、没有命名、没有标注。前端只能照着图量颜色、猜间距、自己起类名。

  • 追问 1：这个红色到底是 #FF4D2E 还是 #FF5236？
  • 追问 2：悬停态、加载态、禁用态都没画？
  • 追问 3：列表空了显示什么？
  • 追问 4：这按钮点了去哪？
GOOD · 系统化 Handoff

"打开这份 Figma 你就知道怎么做"

命名清晰、Variables 完整、Components 配齐 Variants、关键交互有标注、切图打包好。前端 0 追问就能动手。

  • 设计 Tokens：颜色 / 字号 / 间距 / 圆角全部建库
  • Components：所有状态画齐，含 Properties
  • Annotations：跳转、校验、排序、异常态全标注
  • Assets：SVG / @1x@2x@3x 命名一致打包

§03 · GIFT 02 / 8

完整 Variables
(Design Tokens)

承接 C0 / C1 Tokens。颜色、字号、间距、圆角全部建 Variables，并配好 Light / Dark / Brand 三套 Modes。

02
Gift · Design Variables

每个数字都有名字，每个名字都有理由

把"Hex 字符串"换成"语义名"。这样的设计稿在切换深色模式、做品牌换肤时只需要切一个 Mode，前端在代码里也只是切一行 CSS Variable。

§04 · GIFT 03 / 8

完整 Components + Variants

承接 A2 组件。每个 Variants 状态都要齐全（default / hover / active / disabled / loading），并且加上 Properties 让前端能一一对应到 Props。

03
Gift · Components & Variants

"完整"不只是默认态那一张图

真实产品里，组件 80% 的时间不在 default。鼠标悬停、加载中、禁用、错误 —— 这些状态如果你不画，前端写出来的 UI 就只有半成品。

§05 · GIFT 04 / 8

关键标注

间距、行高、字号 Figma 自动展示。但业务逻辑不会自己出现 —— 跳转、校验、排序、异常态都要你写下来。

04
Gift · Behavior Annotations

外形 Figma 替你说，逻辑得你自己说

这个 button 点了去哪？这个 input 校验规则是什么？这个 list 怎么排序？这些信息散在你脑子里，前端没有读心术。

§07 · GIFT 06–08 / 8

文档 · 响应式 · 动画

最后三件礼物：写一份"为什么这么设计"的文档；至少给三档断点；动画规格写到 ms 级别。

Gift · 06

设计文档

Figma 上画的是"是什么"，文档要写的是"为什么"和"什么场景下用"。一两段就够，关键是有。

i使用场景

用于结账流程的最后一步，明确告知用户"提交后将扣款"。

!设计意图

使用强 CTA + 明显视觉权重，是因为转化率上是漏斗的最后一环。次按钮"返回购物车"不能太弱，要能让犹豫的用户找到回路。

↗跟其他模块的关系

⟵ 上游 cart/summary 页面的"去结账" CTA 触发本页
⟶ 下游 提交成功 → /order/success；失败 → 错误 toast，留在本页

Gift · 07

响应式断点

至少三档：375 / 768 / 1440。每档画一遍，前端就知道你的内容怎么塌缩、怎么换行、什么时候用列布局。

375
mobile · sm
768
tablet · md
1440
desktop · lg

小诀窍：响应式不只是宽度变了。导航栏在 mobile 是抽屉、tablet 是 tab、desktop 是横向条 —— 这种"形态切换"必须画。 字号、行数、CTA 文案、列数也都可能在不同断点里变。

Gift · 08

动画规格

动画不是"帅就完事"。前端需要的三个数：duration（多久）、easing（怎么动）、触发条件（什么时候动）。

duration 600ms easing cubic-bezier(.34,1.56,.64,1) trigger 用户点击 / 按下 Tab property transform: translate + scale delay 0ms
