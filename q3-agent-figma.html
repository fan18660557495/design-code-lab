01 / 对比

普通 AI 是一问一答
Agent 是完整闭环

Agent ≠ 更聪明的 AI。Agent = AI + 工具 + 规划 + 反思。 你不再"问问题等答案"，而是下达任务等结果。

old · chat
普通 AI · 一问一答
linear · one-shot
一问 → 一答 → 完
复杂任务需要你拆解、自己调度
new · agent
Agent · 完整闭环
thinking → tool → observe → reflect
下达目标 → 自己闭环 → 整合交付
AI + 工具 + 规划 + 反思 全自动循环
关键 Agent 不是另一个 AI 模型 —— 是 "AI + 一套机制"。 AI 模型是引擎，agent 框架是车架 + 方向盘 + 后视镜 + 自我反思。
02 / 内核

Agent 的
4 件套

不管哪家的 agent 框架, 拆开看就 4 件套 —— tools + planning + memory + reflection。 缺一件就不是 agent, 只是会用工具的 chatbot。

点周围 4 个节点看说明

Agent 内核

中心是 AI 模型（引擎），周围 4 件套是让它成为 agent 的机制。 tools 让它动手，planning 让它规划，memory 让它记忆，reflection 让它自查。

四件齐全才叫 agent
所以 Claude Code / Cursor / Devin 都是 agent —— 它们都有 tools (MCP) + planning (todo) + memory (context) + reflection (verify-then-retry)。 普通 ChatGPT 网页版 不是 agent · 只是 chat。
03 / 模式

多 Agent
3 种协作方式

一个 agent 干不完的活, 派给团队 —— 看 main agent 怎么组织 sub-agent 工作。 3 种 pattern 各有应用场景。

pattern · 01
Orchestrator
一对多 · 指挥型
main agent 拆任务、派给 N 个 sub-agent、收集结果。
适合：复杂目标 · 子任务有依赖
pattern · 02
Pipeline
A → B → C → D · 流水线
A 输出给 B 接，B 给 C，C 给 D。串行 · 每步专精。
适合：工作流明确 · 每步专精
pattern · 03
Parallel
同时跑独立任务 · 并行
N 个 agent 同时启动，跑独立子任务、最后汇总。
适合：子任务独立 · 想省时间
注意 Claude Code 里你能直接用 Task tool 派 sub-agent — 一行 prompt 就够。 不需要自己写 orchestrator 框架。
04 / 派活

main 派活 +
sub 自己闭环

左边看 main 怎么派给 sub，每个 sub 挂哪些工具。
右边看 单个 sub 内部怎么 thinking → tool → observe → reflect 循环。

tree · 派活

main 派给 4 个 sub-agent

每个 sub 挂自己的 MCP / skill 工具
loop · 闭环

单个 agent 自己转

同样的循环 main 也在跑 · 嵌套
领悟 你只下 一个目标 给 main，main 把活拆给 sub，每个 sub 自己跑 think → tool → observe → reflect 闭环。 没人写过调度代码 —— 这就是 agent 框架的价值。
